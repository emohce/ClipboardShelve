# Change Log Format

## 目标
- 统一任务完成后的变更摘要、交付说明与知识沉淀说明。

## 最终输出必须包含
- 变更摘要
- 关键文件
- 风险 / 兼容性影响
- 验证状态
- 知识沉淀状态

## 推荐模板
```md
# 变更摘要
- ...

# 关键文件
- [path/to/file#L1](path/to/file#L1): 作用

# 风险 / 兼容性影响
- ...

# 验证状态
- 已执行: ...
- 未执行: ...

# 知识沉淀状态
- 命中历史记录: ...
- 新增 / 更新 Error Memory: ...
- 新增 / 更新 ADR: ...
- 新增 / 更新 Glossary: ...
```

## 规则
- 变更摘要只写本次真实改动。
- 关键文件优先列核心文件，不罗列所有附带修改。
- 风险部分必须说明未验证项、兼容边界、环境限制。
- 如果命中了历史错误记录，必须明确记录编号和采用 / 规避策略。
- 如果本次没有新增知识沉淀，也要明确写“无”。
