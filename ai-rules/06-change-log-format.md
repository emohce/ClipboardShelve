# Change Log Format

## 目标
- 统一任务完成后的变更摘要、交付说明与后续协作格式。

## 最终输出必须包含
- 变更摘要
- 关键文件
- 风险/兼容性影响
- 验证结果
- 后续可选优化

## 推荐模板
```md
# 变更摘要
- ...

# 关键文件
- path/to/file: 作用

# 风险/兼容性影响
- ...

# 验证结果
- `npm run build`: pass
- 手工验证: ...

# 后续可选优化
- ...
```

## 编写规则
- 变更摘要只写本次真实改动，不复述背景。
- 关键文件优先列核心文件，不罗列所有附带修改。
- 风险部分必须说明未验证项、兼容边界、环境限制。
- 若无后续优化必要，可写“无”。

## 提交信息建议
- 遵循 Conventional Commits。
- 推荐格式：`type(scope): summary`
- 文档类优先使用 `docs(...)`，流程规范类可用 `docs(workflow): ...`
