# Feature: 移动效果优化

## 1. 目标

优化列表键盘导航的滚动策略和选中效果，解决删除/锁定状态下的渲染异常，确保移动流畅、边界处理优雅。

## 2. 背景

当前列表键盘导航存在以下问题：
- 长按上下键采用逐项强行居中策略，与分页逻辑不统一，导致滚动跳跃感强
- 单步移动时无论元素是否可见都触发居中滚动，造成不必要的视觉跳动
- 删除 item 后选中效果错位，有时选中的是已被删除的元素或位置计算错误
- 锁状态切换后图标渲染延迟，需要移动选中后才更新，且图标换行影响布局
- 列表项渲染性能有待优化，大数据量下可能出现卡顿

## 3. 用户场景 / 使用场景

### 场景 1：长按快速浏览
用户长按 ↑/↓ 键快速浏览大量历史记录，期望按页跳转并居中展示当前选中项，快速定位目标内容。

### 场景 2：单步精确定位
用户单击 ↑/↓ 键逐个查看 item，期望当前 item 已可见时不触发滚动，仅在即将移出可视区域时才平滑滚动，保持视觉稳定。

### 场景 3：删除后继续操作
用户删除当前 item 后，期望自动选中下一个 item（或上一个，如果是最后一个），选中状态正确、页面不闪烁，可立即继续操作。

### 场景 4：锁定状态确认
用户按锁定快捷键后，期望立即看到锁图标状态变化，确认操作生效，无需再移动一次才看到反馈。

### 场景 5：密集列表浏览
用户在收藏夹等可能包含密集锁/收藏图标的列表中浏览，期望图标不挤占内容空间，长文本自动缩略展示。

## 4. 非目标

- 不修改鼠标悬浮/点击交互逻辑
- 不修改多选模式的批量操作逻辑
- 不修改图片/文本预览功能
- 不引入新的动画库或复杂的过渡效果
- 不改变现有的虚拟滚动实现（vue-virtual-scroller）

## 5. 验收标准

### 5.1 长按分页滚动
- [ ] 长按 ↑/↓ 超过 300ms 后，按页步进（pageStep = floor(容器高度 / 44px * 0.9)）
- [ ] 每次跳转后选中项居中展示（block: "center"）
- [ ] 滚动间隔随长按时间加速，最小间隔不低于 50ms
- [ ] 边界处（首/尾 item）长按不再触发无意义的滚动尝试

### 5.2 单步可见性滚动
- [ ] 单击 ↑/↓ 移动一个 item
- [ ] 若目标 item 已在可视区域内（isNodeFullyVisible 返回 true），不触发滚动
- [ ] 若目标 item 部分或完全不可见，滚动使其可见（block: "nearest"）
- [ ] 单步移动后选中效果（高亮样式）必须正确显示

### 5.3 删除后选中修复
- [ ] 删除当前 item 后，自动选中下一项（同一索引位置的新 item）
- [ ] 若删除的是最后一项，选中前一项
- [ ] 选中状态变化时页面不割裂、不闪烁
- [ ] 删除后 `activeIndex` 与实际渲染的选中项一致

### 5.4 锁状态即时更新
- [ ] 按锁定快捷键后，锁图标状态立即更新（不依赖移动选中）
- [ ] 批量锁定时，所有选中项的图标同时更新
- [ ] 图标渲染不触发异常换行，内容区域被挤占时自动缩略（text-overflow: ellipsis）

### 5.5 性能要求
- [ ] 单行渲染逻辑独立，不影响列表整体性能
- [ ] 300ms 内完成渲染（肉眼无感知延迟）
- [ ] 筛选/搜索状态下移动不导致列表错乱

## 6. 边界与异常

| 场景 | 预期行为 |
|------|----------|
| 列表为空 | 导航操作无响应，不报错 |
| 首项按 ↑ | 保持在首项，滚动到顶部（block: "start"） |
| 末项按 ↓ | 触发 loadMore，有新数据后自动移动到新数据首项 |
| loadMore 无数据 | 保持在末项，滚动到底部（block: "end"） |
| 快速连续按方向键 | 每次按键都响应，不丢失操作 |
| 删除时列表只剩一项 | 删除后列表为空，activeIndex 设为 0 |
| 筛选/搜索状态下删除 | 从筛选后的列表中删除，选中状态正确维护 |
| 多选模式下删除 | 批量删除后，退出多选模式，选中剩余项的第一个 |

## 7. 影响范围

### 7.1 组件
- `src/cpns/ClipItemList.vue` - 核心导航逻辑、长按检测、滚动策略
- `src/cpns/ClipItemRow.vue` - 选中样式、锁/收藏图标渲染

### 7.2 样式
- `src/style/cpns/clip-item-row.less` - 图标布局、缩略样式、选中效果

### 7.3 全局
- `src/global/hotkeyBindings.js` - 可能涉及的快捷键绑定调整

### 7.4 数据流
- `activeIndex` 维护逻辑
- `showList` 变化后的选中恢复逻辑
- `pendingHighlightedItemId` / `pendingActiveIndexAfterDelete` 修复
