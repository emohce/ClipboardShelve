# Definition of Done

## 目标
- 统一定义任务“真正完成”的最低标准，避免只改完代码就结束。

## 必须满足
- 代码改动已经完成，并与当前批准范围一致。
- 已说明关键影响文件、影响范围、兼容边界和残余风险。
- 已明确验证状态，区分“已检查”“未检查”“仅代码推断”。
- 已说明是否命中历史错误记录，以及采用 / 规避了什么。
- 已判断是否需要新增或更新 `Error Memory`、`ADR`、`Glossary`。
- 最终说明已附关键文件的 Markdown 链接，并尽量带行号。

## 复杂任务额外要求
- `spec`、`plan`、`tasks`、`verify` 已同步。
- 高风险目录改动已说明影响与回退点。
- 如存在未验证项，已明确写出，不默认视为通过。

## 不满足的典型情况
- 只描述改了什么，没有说明风险和未验证项。
- 命中了历史失败模式，但没有说明这次如何规避。
- 已产生新的规则或经验，却没有落盘。
