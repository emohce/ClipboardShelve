# Feature: 列表移动与删除修复

## 1. 目标
- 修复主列表在键盘上下移动、长按翻页、删除后恢复选中时的错位、闪烁与滚动异常。
- 保证锁定状态与收藏状态在列表中即时反映，不依赖再次移动才刷新。
- 统一键盘移动相关的滚动与高亮逻辑，保证用户始终能明确看到当前选中行。
- 保持现有 Vue 3 + JavaScript + 虚拟列表方案，不新增依赖、不改目录结构。

## 2. 背景
- 原始需求见 [../BjVkZ-zt-move.md](../BjVkZ-zt-move.md)。
- 当前 `ClipItemList` 同时存在单步导航、长按自动滚动、分页滚动等多套路径，行为不一致。
- 单步移动时偶发高亮不可见或缺失，与滚动策略及虚拟列表可见区域判断有关。
- 当前删除后高亮索引可能由 [src/views/Main.vue](../../../src/views/Main.vue) 与 [src/cpns/ClipItemList.vue](../../../src/cpns/ClipItemList.vue) 共同维护，存在互相覆盖风险。
- 当前锁定切换未及时驱动展示列表更新时，筛选视图下可能出现延迟。

## 3. 用户场景 / 使用场景
- 用户单击 `ArrowUp` / `ArrowDown` 时：若当前项在视口内完整可见，则仅更新高亮、不主动滚动；若不可见（含部分被裁切），则滚动至可见。
- 用户长按 `ArrowUp` / `ArrowDown` 时：在首次有效单步之后进入分页式跳转，复用现有「分页并居中」能力，目标项居中且带有明确选中样式，避免逐项连续居中造成的抖动。
- 用户删除当前高亮项后：若其后仍有项则选中下一项；若为最后一项则选中上一项；列表不出现明显闪烁、错位或割裂渲染。
- 用户在 `locked` 筛选或涉及收藏/锁图标的场景下切换状态时：行内图标与列表数据尽快一致；图标区域在宽度紧张时以缩略方式展示，避免异常换行拖累布局。
- 用户在搜索/筛选后继续移动选中：高亮与可见项一致，不因多套触发源重复滚动或冲突。

## 4. 非目标
- 不重做全局快捷键体系。
- 不替换 TanStack Virtual。
- 不改动 `public/plugin.json`、`public/preload.js`、`public/listener.js`。
- 不在本次 Spec 中约定新的性能监控埋点；仅通过避免无意义重排满足「列表交互可感知流畅」。

## 5. 验收标准
- 单步上下键：`可见则不滚，不可见才滚`；高亮始终可操作且视觉上可辨认。
- 长按上下键：复用分页跳转逻辑，目标项居中展示并保持选中样式。
- 删除后：`activeIndex` 与展示列表一致，持久化数据与界面无错乱；无明显整表闪烁。
- 锁/收藏状态：切换后当前列表内及时更新；`locked` 筛选下解锁导致项消失时，高亮索引安全收敛。
- 列表构建命令 `pnpm run build` 通过。

## 6. 边界与异常
- 列表为空、仅 1 条、删除至 0 条时不抛错、不出现非法索引。
- 长按滚动触及列表底部：仅在仍可加载更多时继续推进；否则停在最后一项。必要时可采用「半页滚动」作为兜底以减少卡在倒数几行时的体验劣化（实现阶段由 Plan 取舍）。
- `collect` 标签页保留既有删除/取消收藏约束，不改为破坏性删除。
- 多种移动入口（单击键、长按、鼠标等）不得在同一时刻产生互相覆盖的重复滚动调用；需收敛到单一底层导航/滚动路径。

## 7. 影响范围
- [src/cpns/ClipItemList.vue](../../../src/cpns/ClipItemList.vue)：键盘导航、长按滚动、删除后恢复选中、锁状态与列表刷新协调。
- [src/cpns/ClipItemRow.vue](../../../src/cpns/ClipItemRow.vue)：单行展示、锁/收藏图标区域布局与缩略，避免换行与性能回退。
- [src/views/Main.vue](../../../src/views/Main.vue)：删除回写与父子索引恢复职责收敛。
- [src/style/cpns/clip-item-list.less](../../../src/style/cpns/clip-item-list.less)：列表与行头紧凑布局、高亮与图标区域样式。

## 8. 待确认项
- 长按「分页」步长沿用现有 `scrollByPage` / 虚拟列表行为，不单独新增配置文件项，除非实现中发现步长与产品预期明显不一致。
- 「300ms 内渲染成功」以不引入明显同步重排、不阻塞主线程长任务为原则验收，不在此 Spec 中绑定具体性能探针数值。
