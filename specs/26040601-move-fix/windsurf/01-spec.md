# Feature: 列表移动效果优化

## 1. 目标

优化剪贴板列表主界面的键盘移动体验，包括：
- 长按上下键实现分页滚动并居中展示选中项
- 单步移动遵循"可见则不滚，不可见才滚"原则
- 修复删除 item 后选中效果错乱问题
- 修复锁状态更新不及时问题
- 修复锁/收藏图标渲染异常换行问题

## 2. 背景

当前列表移动存在以下问题：
1. 长按上下键只是重复触发单步移动，导致逐项强行居中，体验不佳
2. 单步移动使用 `block: "nearest"`，部分可见时仍可能滚动，不符合用户预期
3. 删除 item 后选中效果可能出现错位、数据混乱、渲染闪烁
4. 锁状态（locked/collected）更新不及时，需等待移动选中后才更新
5. 锁/收藏图标渲染时可能触发异常换行，挤压内容

## 3. 用户场景 / 使用场景

### 3.1 长按上下键分页滚动
- 用户在列表中长按向上/向下键
- 列表按屏幕高度分页滚动
- 选中项在分页后居中展示
- 选中效果（样式和标记）正常显示

### 3.2 单步移动可见性控制
- 用户单击向上/向下键
- 若目标项完全可见，不滚动
- 若目标项不可见（包括一半可见），滚动到可见
- 选中效果正常显示

### 3.3 删除 item 后选中行为
- 用户删除当前选中的 item
- 自动选中上一条（若删除的是第一条则选中下一条）
- 页面渲染不割裂闪烁
- 选中效果不错位

### 3.4 锁状态及时更新
- 用户切换锁/收藏状态
- 状态立即更新到数据库
- 图标不触发异常换行
- 内容挤压时图标自动缩略展示

## 4. 非目标

- 不改变现有的鼠标悬浮选中效果
- 不改变现有的虚拟滚动基础架构
- 不改变现有的快捷键绑定配置
- 不涉及其他页面（如设置页）的列表

## 5. 验收标准

### 5.1 长按上下键分页滚动
- 长按向上键：列表向上滚动一屏高度，选中项居中
- 长按向下键：列表向下滚动一屏高度，选中项居中
- 分页步长基于屏幕高度计算（`scrollParentRef.clientHeight * 0.9 / estimateItemSize`）
- 选中效果（高亮样式、标记）正常显示

### 5.2 单步移动可见性控制
- 单击向上/向下键：目标项完全可见时不滚动
- 目标项一半可见时判定为不可见，滚动到完全可见
- 目标项完全不可见时滚动到可见
- 选中效果正常显示

### 5.3 删除 item 后选中行为
- 删除中间项：选中上一条
- 删除第一条：选中下一条（原第二条）
- 删除最后一条：选中上一条（原倒数第二条）
- 删除后不出现选中效果错位
- 删除后页面渲染不闪烁

### 5.4 锁状态及时更新
- 切换锁/收藏状态后立即调用 `window.queuePersistDb()`
- 图标渲染不触发换行
- 内容挤压时图标自动缩略展示
- 单行渲染性能在 300ms 内

### 5.5 边界条件
- 列表为空时不触发移动
- 列表只有一项时删除后不选中
- 不同 tab（全部/文本/图片/文件/收藏）行为一致
- 搜索面板展开时行为一致

## 6. 边界与异常

### 6.1 空列表
- 列表为空时不响应键盘移动
- 不触发滚动操作

### 6.2 单项列表
- 只有一项时删除后不选中任何项
- 长按上下键不触发分页滚动

### 6.3 懒加载场景
- 接近列表末尾时触发懒加载
- 懒加载后保持选中状态
- 懒加载失败时不影响选中效果

### 6.4 并发操作
- 删除与移动同时发生时，以删除后的数据为准
- 锁状态更新与移动同时发生时，状态更新优先

### 6.5 虚拟滚动兼容性
- 修改滚动逻辑不影响 TanStack Virtual 的虚拟滚动
- 保持虚拟滚动的性能优势

## 7. 影响范围

### 7.1 主要文件
- `src/cpns/ClipItemList.vue`：列表组件，键盘导航和滚动逻辑
- `src/hooks/useVirtualListScroll.js`：虚拟滚动 hook，可见性判断和滚动方法
- `src/hooks/useListNavigation.js`：列表导航 hook，选中状态管理
- `src/views/Main.vue`：主页面，删除操作和列表数据管理

### 7.2 次要文件
- `src/global/hotkeyRegistry.js`：快捷键注册（可能需要调整长按检测）
- `src/cpns/ClipItemRow.vue`：列表行组件，图标渲染优化

### 7.3 样式文件
- `src/style/` 下的相关 Less 文件：图标缩略样式
