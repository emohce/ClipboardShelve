# Feature: 剪贴板预览效果优化（图片 / 文字 / Shift 交互）

## 1. 目标

- 优化插件内**图片预览**的布局与滚动：在保持现有「占屏比例与展示策略」总体不变的前提下，矮图在可视区内**垂直居中**；窄图在宽度方向**放大至可用宽度**（必要时出现横向滚动），优先**单屏内完整展示**可一屏容纳的内容；特高图以**宽度铺满**为基准居中展示，通过 **Shift + 上/下键**在预览容器内**滚动**（明确不是缩放）。
- 优化**文字预览**（Shift 长按触发的长文本层）：**单行**时样式与图片预览意图一致——约 **90% 窗口宽度**、**垂直居中**、单行展示，超出部分用省略号（` ...`），**不自动换行**；**多行**时在同一预览层内用 **Shift + 上/下键**滚动内容（不是缩放）。
- 在 **Shift 长按已打开预览**的期间，**阻断仅由鼠标移动**引起的「预览目标切换」（例如列表高亮随鼠标换行导致预览跟着换条）；**松开 Shift** 后恢复正常的鼠标与列表联动。

## 2. 背景

- 当前实现集中在列表组件内：图片与长文本预览、`Shift` 长按触发预览、悬浮预览配置等均在 [`src/cpns/ClipItemList.vue`](../../src/cpns/ClipItemList.vue) 及相关样式中。
- 图片预览容器当前为顶部对齐（`align-items: flex-start`），矮图在遮罩内视觉上偏上；文字预览使用 `pre-wrap` 与自动换行，与「单行铺满 + 省略」的期望不一致。
- `activeIndex` 在鼠标移入行时会更新，且在 `Shift` 长按预览开启时会触发「随当前行刷新预览」的逻辑，导致鼠标划过列表即可切换预览对象，与「预览中锁定目标直至松开 Shift」冲突。

## 3. 用户场景 / 使用场景

- **场景 A（图片·矮图）**：用户 `Shift` 长按打开一张高度小于视区的图片，期望图片在黑色遮罩内**上下居中**，左右保持现有居中逻辑，观感不「贴顶」。
- **场景 B（图片·窄图）**：用户预览一张宽度远小于窗口的图片，期望图片在宽度上**放大到可用宽度**（或直至需要横向滚动），仍可用 **Shift + 上/下键**在容器内滚动查看溢出部分（与缩放区分：不改变缩放比例以外的交互语义时，以滚动为主）。
- **场景 C（图片·长图）**：用户预览明显高于视区的图片，期望以**宽度适配**为主、水平居中，用 **Shift + 上/下键**纵向滚动查看（非缩放）。
- **场景 D（文字·单行）**：剪贴项为单行或可视作单行展示时，预览层**宽约 90%**、**垂直居中**，过长用 ` ...` 截断，**不换行**。
- **场景 E（文字·多行）**：长文本多行展示时，在预览层内 **Shift + 上/下键**滚动阅读。
- **场景 F（Shift 预览中）**：用户按住 `Shift` 查看某条预览时，鼠标在列表上移动**不应**把预览切换到鼠标下方条目；**松开 `Shift`** 后，再移动鼠标或操作列表，预览目标可按现有规则更新。

## 4. 非目标

- 不改变 uTools 插件入口、全局快捷键注册方式、[`public/plugin.json`](../../public/plugin.json) / preload 等与预览无直接关系的契约（除非验证中发现硬性依赖）。
- 不将本次需求扩展为「重做整个预览架构」或引入新依赖（如新富文本引擎）。
- 不将「鼠标滚轮 / 触控板」在预览内的行为列为本次必改项（除非与 Shift+方向键冲突需统一说明）。
- 不要求修改「设置页」中悬浮预览开关与延迟配置的语义，仅当与 Shift 锁定逻辑冲突时在实现阶段再对齐。

## 5. 验收标准

- **图片**
  - 在常见窗口尺寸下，**明显矮于视区**的图片在预览遮罩内容区中**垂直居中**（可接受 ± 与 padding 一致的像素级偏差）。
  - **明显窄于可用宽度**的图片：在保持「不强行拉伸失真」的前提下，展示结果满足「宽度方向放大至合理上限、必要时横向滚动」；与原始需求「占据比例不用改」一致时，以**不改变既有缩放/适配算法的大方向**为前提做布局修正。
  - **高度远大于视区**的图片：以宽度方向适配为主、水平居中；**Shift + 上/下键**仅引起预览内容区**滚动**，无可感知的「缩放」副作用。
- **文字**
  - **单行**（按产品规则判定为单行展示时）：内容区域约 **90% 窗口宽度**、**垂直居中**；超出显示宽度时末尾为省略号样式（含 ` ...` 语义），**不出现自动换行**。
  - **多行**：**Shift + 上/下键**可滚动正文区域，行为为滚动而非缩放。
- **Shift 预览锁定**
  - 在 `Shift` 长按导致的预览**显示期间**，用户仅移动鼠标经过其他列表行，**预览内容与对应条目保持不变**（不因鼠标移动切换预览对象）。
  - 松开 `Shift` 后，列表与预览的交互恢复为当前产品既有设计（含悬浮预览配置、键盘导航等）。

## 6. 边界与异常

- 窗口尺寸变化、高分屏、极窄或极矮窗口：预览仍不应崩溃；居中与滚动条表现可降级为「可滚动即可」，但需在 [`04-verify.md`](04-verify.md) 中记录实测窗口范围。
- 图片加载失败、非可解码资源：保持现有错误提示路径，不因布局调整而丢失。
- 多选模式、搜索框聚焦、方向键挂起悬浮预览等**既有**行为与本次改动交叉时，以不破坏原有「挂起 / 恢复」语义为原则；若冲突，在实现阶段在 `02-plan.md` 中写明取舍。
- 与 **悬浮预览**（非 Shift）同时存在时：本次需求以 **Shift 长按预览**与「预览中鼠标不切换」为主；悬浮预览是否需同样「锁定」不在本 spec 强制范围，若需一致则在 `待确认项` 中决策。

## 7. 影响范围

- **主要**：[`src/cpns/ClipItemList.vue`](../../src/cpns/ClipItemList.vue) — 图片/文字预览状态、`showImagePreview` / `showTextPreview`、`handleMouseOver` 与 `activeIndex` 联动、`keyboardTriggeredPreview` 相关 watch、图片/文字预览键盘滚动逻辑。
- **样式**：同文件 scoped 样式及/或 [`src/style/cpns/clip-item-list.less`](../../src/style/cpns/clip-item-list.less) 中与 `.image-preview-modal`、`.text-preview-modal` 相关的布局与滚动。
- **快捷键**：若 Shift+方向键滚动需与列表导航注册 feature 协调，可能涉及同文件内 `registerFeature("text-preview-scroll-*")` 等（以实际代码为准）。
- **文档**：本 feature 目录下后续 `02-plan.md`、`03-tasks.md`、`04-verify.md`；不涉及改用户面向说明文档，除非产品要求同步。

## 8. 待确认项

1. **「单行文字」判定**：是否仅指「不含换行且长度 ≤ 当前长文本阈值」的文本，还是所有在预览层应以单行展示的 case（可能涉及是否对短文本也弹出 Shift 文字预览）。当前代码中长文本与短文本分支以 `LONG_TEXT_THRESHOLD` 与换行判断为界。
2. **「宽度小则自动扩放」的上限**：放大后若仍宽于视区，是否以「横向滚动」为唯一手段（与「单页优先」的平衡点）。
3. **外部独立预览窗口**（若仍存在 `openExternalPreview` 路径）：本次是否纳入与插件内层一致的布局与交互，或明确仅规范插件内 `Teleport` 预览层。
