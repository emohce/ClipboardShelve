# Feature: 剪贴板预览效果优化（图片 / 文字 / Shift 锁定）

## 1. 目标
- 优化图片预览：矮图在预览区上下居中，窄图优先按可用宽度放大，长图按宽度铺满并支持滚动浏览。
- 优化文字预览：单行文本按约 90% 窗口宽度展示、上下居中、单行省略；多行文本支持滚动阅读。
- 优化 Shift 预览交互：Shift 长按预览期间，鼠标移动不切换预览对象；松开 Shift 后恢复现有联动。

## 2. 背景
- 原始需求见 [`docs/todo/260405-交互优化/预览效果优化/zt-Preview-原始需求.md`](../../docs/todo/260405-交互优化/预览效果优化/zt-Preview-原始需求.md)。
- 当前预览主逻辑集中在 [`src/cpns/ClipItemList.vue`](../../src/cpns/ClipItemList.vue)。
- 图片预览容器当前在 [`src/cpns/ClipItemList.vue`](../../src/cpns/ClipItemList.vue) 中使用顶部对齐，导致矮图视觉上贴顶。
- 文字预览当前使用 `pre-wrap` 与自动换行，不符合单行文本“铺满宽度 + 省略”的目标。
- 当前 `activeIndex` 会在鼠标移入时更新，而 `keyboardTriggeredPreview` 激活期间会基于 `activeIndex` 刷新预览，导致 Shift 预览时鼠标划过列表会切换预览对象。

## 3. 用户场景 / 使用场景
- 场景 1：用户预览一张高度较小的图片，希望图片在遮罩内容区内上下居中，而不是贴在顶部。
- 场景 2：用户预览一张宽度较小的图片，希望图片尽量放大到可用宽度；当横向仍有溢出时，可继续滚动查看。
- 场景 3：用户预览一张高度很大的长图，希望以宽度铺满为主、水平居中，并通过 `Shift + ↑/↓` 浏览完整内容。
- 场景 4：用户预览单行长文本，希望文本在预览层内接近全宽展示、上下居中、不换行，超出部分显示省略效果。
- 场景 5：用户预览多行文本，希望保留原文内容，并通过 `Shift + ↑/↓` 在预览容器内滚动阅读。
- 场景 6：用户按住 `Shift` 预览当前项时，鼠标只是经过其他列表项，预览仍保持当前对象，直到松开 `Shift`。

## 4. 非目标
- 不重做预览架构，不新增预览类型，不引入新依赖。
- 不改变现有预览触发方式，`Shift` 长按触发机制保持现状。
- 不修改 [`public/plugin.json`](../../public/plugin.json)、[`public/preload.js`](../../public/preload.js)、[`public/listener.js`](../../public/listener.js) 与 `src/global/` 之外的运行时契约。
- 不纳入 [`src/cpns/ClipItemList.vue`](../../src/cpns/ClipItemList.vue) 中 `openExternalPreview` 外部窗口预览行为。
- 不把鼠标滚轮、触控板手势列入本次必做项。

## 5. 验收标准
- 图片预览
  - 矮图在预览内容区内上下居中展示，不再默认贴顶。
  - 窄图在不破坏纵横比的前提下优先按可用宽度放大；能单屏看完时优先单屏展示。
  - 长图以宽度适配为主、水平居中，`Shift + ↑/↓` 只触发纵向滚动，不触发缩放。
  - 图片横向溢出时支持 `Shift + ←/→` 横向滚动查看。
- 文字预览
  - 单行文本最大展示宽度约为窗口宽度的 90%，上下居中，禁止自动换行，超出部分显示省略效果。
  - 多行文本保留原文内容，`Shift + ↑/↓` 可以在文本预览容器内滚动。
- Shift 锁定
  - Shift 长按预览显示期间，仅鼠标移动不会改变当前预览对象。
  - 松开 Shift 后，当前既有的鼠标悬浮高亮、hover preview 挂起/恢复逻辑继续按原规则工作。

## 6. 边界与异常
- 空列表、空文本、无效图片源时，不应强行打开预览；保持当前安全退出路径。
- 图片加载失败时，继续显示现有错误提示，不因布局调整丢失失败态。
- 文件类型中复用图片预览入口的场景沿用同一套图片预览规则。
- 多选状态、搜索输入聚焦、hover preview 挂起/恢复、键盘导航等既有语义不能回归。
- `Shift + 方向键` 在预览打开时应优先服务预览滚动，不应误触发列表导航。
