# Feature: Preview效果优化

## 1. 目标
- 优化图片预览的展示效果：支持自适应居中、智能扩放、滚动浏览
- 优化文字预览的展示效果：单行铺满展示、多行滚动浏览
- 优化预览状态下的交互体验：防止鼠标误触切换预览内容

## 2. 背景
当前预览功能已实现基础能力，但在展示效果和交互体验上存在优化空间：
- 图片预览时小图片不能自动扩放，高图片不能有效浏览
- 文字预览时单行内容没有最大宽度利用，过长时自动换行影响阅读
- 按住Shift预览时，鼠标移动会意外触发预览切换，打断浏览体验

## 3. 用户场景 / 使用场景

### 场景1：预览小尺寸图片
用户复制了一个小图标或截图，希望在预览窗口中能看到清晰的放大效果，而不是保持原小尺寸显示。

### 场景2：预览长截图/高图片
用户复制了一个长网页截图或手机截图，希望在预览窗口中以宽度铺满方式查看，并能通过快捷键滚动浏览完整内容。

### 场景3：预览单行长文本
用户复制了一条长链接或单行日志，希望在预览窗口中以最大宽度展示（90%窗口宽度），如果超长则用缩略符截断显示，不换行。

### 场景4：预览多行文本
用户复制了一段代码或多行日志，希望在预览窗口中查看完整内容，并支持快捷键滚动浏览。

### 场景5：稳定预览不中断
用户按住Shift预览某条内容时，希望鼠标移动不会意外切换到其他条目，保持当前预览稳定显示。

## 4. 非目标
- 不改变预览窗口的整体尺寸和位置
- 不引入新的预览类型（仅优化现有图片/文字预览）
- 不修改外部窗口预览的行为
- 不修改预览的触发方式（Shift长按触发保持不变）

## 5. 验收标准

### 5.1 图片预览优化
| 条件 | 预期行为 |
|------|---------|
| 图片高度较小 | 自动上下居中展示，保持比例 |
| 图片宽度较小 | 自动扩放至窗口可用宽度，直至达到最大宽度限制 |
| 图片高度很高 | 以宽度铺满为准，图片上方居中，支持滚动查看下方内容 |
| Shift+↑/↓ | 每次滚动100px，平滑滚动浏览图片 |

### 5.2 文字预览优化
| 条件 | 预期行为 |
|------|---------|
| 单行文本 | 铺满90%窗口宽度，不换行，超长时用` ...`缩略显示 |
| 多行文本 | 支持Shift+↑/↓键滚动浏览，每次滚动100px |
| 文本容器 | 保持半透明黑色背景，圆角8px |

### 5.3 预览状态交互
| 条件 | 预期行为 |
|------|---------|
| 按住Shift预览时 | 阻断鼠标移动导致的预览切换 |
| 松开Shift后 | 恢复正常鼠标移动切换预览逻辑 |

## 6. 边界与异常

### 6.1 边界情况
- 极小图片（<50px）：至少扩放到200px宽度以保证可见性
- 极宽图片（>屏幕宽度）：限制最大宽度为窗口宽度的95%
- 极高图片（>屏幕高度3倍）：提供视觉提示（如滚动条或箭头）表明可滚动

### 6.2 异常处理
- 图片加载失败：保持现有错误提示，不应用扩放逻辑
- 空文本内容：不触发预览
- 快捷键冲突：Shift+方向键仅当预览激活时拦截，不影响其他组件

## 7. 影响范围

### 7.1 修改文件
- `src/cpns/ClipItemList.vue`：主要修改点
  - 修改 `showImagePreview` 函数的图片样式逻辑
  - 修改 `showTextPreview` 函数的文本容器样式
  - 修改 `handleImagePreviewKeydown` 的滚动处理
  - 修改鼠标移动处理逻辑，添加预览状态判断

### 7.2 数据/状态变化
- `imagePreview.imageStyle`：新增动态计算的最大宽度/高度约束
- `textPreview.style`：调整宽度为90%，添加单行溢出处理
- 新增预览状态标志：用于判断当前是否处于预览模式

### 7.3 交互变化
- 图片预览的视觉表现：从固定max-height改为智能扩放
- 文字预览的文本处理：从自动换行改为单行截断
- 鼠标移动行为：预览激活时阻断切换，Shift释放后恢复

### 7.4 外部依赖
- 依赖现有 `vue-virtual-scroller` 的滚动机制（无变化）
- 依赖现有 Shift 键检测逻辑（无变化）
