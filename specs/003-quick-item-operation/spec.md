# Feature Specification: 单条目快捷操作增强

**Feature Branch**: `003-quick-item-operation`  
**Created**: 2026-04-09  
**Status**: Draft  
**Input**: User description: "/speckit-specify 新增一个任务: `260409-quickItemOperation`"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 单条目别名快速新增与更新 (Priority: P1)

作为日常高频使用者，我希望在单个条目上直接通过 F2 完成别名新增或更新，不改变收藏页既有弹出层操作习惯，同时新增可直达的别名逻辑，以减少重复操作。

**Why this priority**: 这是用户高频路径，直接影响单条目管理效率与一致性体验。

**Independent Test**: 在普通列表页与收藏页分别选择单个条目，触发 F2，验证能新增或更新别名；收藏页仍保留原弹出层流程且可走新增别名逻辑。

**Acceptance Scenarios**:

1. **Given** 用户选中一个无别名的单条目，**When** 按下 F2，**Then** 系统应允许新增别名并在保存后立即生效。
2. **Given** 用户选中一个已有别名的单条目，**When** 按下 F2，**Then** 系统应允许更新别名并覆盖旧值。
3. **Given** 用户位于收藏页并选中单条目，**When** 按下 F2，**Then** 原有弹出层流程继续可用，且用户仍可完成别名新增或更新。
4. **Given** 用户通过 F2 打开别名编辑弹窗，**When** 按 Enter，**Then** 当前输入应被保存并立即生效。
5. **Given** 用户通过 F2 打开别名编辑弹窗，**When** 按 Esc，**Then** 应取消本次编辑且不改动已存别名。

---

### User Story 2 - 操作列表与自动执行快捷键对齐 (Priority: P2)

作为键盘操作用户，我希望 `c-right` 展开的操作列表与自动执行快捷键序号保持一致，并把自动执行从 `c-s-num` 调整为 `c-a-num`，以降低记忆负担和误操作。

**Why this priority**: 操作入口与快捷键编号不一致会直接导致执行错误，影响可预期性。

**Independent Test**: 打开任意条目操作列表，使用 `c-right` 查看序号并逐项用 `c-a-num` 触发，验证触发结果与列表序号一致，且 `c-s-num` 不再作为该自动执行入口。

**Acceptance Scenarios**:

1. **Given** 用户已展开单条目操作列表，**When** 查看操作序号并按对应 `c-a-num`，**Then** 应执行与当前序号一致的操作。
2. **Given** 用户按下旧快捷键 `c-s-num`，**When** 尝试自动执行序号操作，**Then** 不应再触发原自动执行逻辑。

---

### User Story 3 - 有别名时的保存快捷触发 (Priority: P3)

作为处理文件型剪贴内容的用户，我希望在存在别名时使用 `s-enter` 快速保存并触发对应行为：单文件场景自动重命名并粘贴到目标位置；其他内容类型保持原组合粘贴逻辑不变。

**Why this priority**: 该能力提升文件型场景的效率，同时要求不破坏现有文本/组合粘贴行为。

**Independent Test**: 分别准备“单文件且有别名”“组合粘贴内容”“纯文本”三种数据，触发 `s-enter`，验证三种场景行为符合预期且互不影响。

**Acceptance Scenarios**:

1. **Given** 当前条目是单文件且已设置别名，**When** 按下 `s-enter`，**Then** 系统应自动按别名完成文件重命名并粘贴到目标位置。
2. **Given** 当前条目是组合粘贴内容或纯文本，**When** 按下 `s-enter`，**Then** 系统应继续执行原组合粘贴流程，不进入单文件重命名逻辑。

---

### Edge Cases

- 当用户未选中条目时按 F2 或 `s-enter`，系统应提示无可操作目标且不改动现有数据。
- 当单文件条目存在别名但重命名目标不可写或冲突时，系统应中止写入并给出可理解提示，不影响原文件内容。
- 当操作列表显示序号与内部可执行动作不一致时，系统应以当前显示序号为准进行校验并阻止错误映射执行。
- 当条目类型在操作过程中发生变化（例如用户切换选择）时，系统应按最新选中条目重新判断逻辑，避免对旧条目执行动作。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须支持在单条目上通过 F2 触发“别名新增或更新”能力。
- **FR-002**: 系统必须在收藏页保留现有弹出层交互路径，同时让用户在该路径下完成别名新增或更新。
- **FR-003**: 系统必须将 `c-right` 展开的操作列表与自动执行序号触发行为保持一致。
- **FR-004**: 系统必须将自动执行序号快捷键从 `c-s-num` 调整为 `c-a-num`。
- **FR-005**: 系统必须在存在别名时支持 `s-enter` 作为快捷触发保存按键。
- **FR-006**: 当条目为“单文件且有别名”时，`s-enter` 必须触发“自动重命名并粘贴到对应位置”行为。
- **FR-007**: 当条目为组合粘贴或纯文本类型时，`s-enter` 必须保持原组合粘贴行为，不触发单文件重命名逻辑。
- **FR-008**: 系统必须确保新旧快捷键迁移后，用户可通过可见操作入口理解当前有效快捷键与动作映射。
- **FR-009**: 系统必须在别名编辑弹窗内提供统一键盘语义：`Enter` 确认保存、`Esc` 取消编辑，并避免误触发主列表回车快捷键。
- **FR-010**: 系统必须在条目列表中支持“别名优先展示 + 原始信息省略展示”；图片条目需在图片左侧展示别名且不影响原图片预览与点击通路。

### Key Entities *(include if feature involves data)*

- **条目 Item**: 用户当前选中的操作对象，关键属性包括条目类型（单文件/组合/文本）、所属页面（普通页/收藏页）与可执行动作集合。
- **别名 Alias**: 绑定到条目的用户定义名称，可被新增、更新，并在特定快捷操作中作为命名依据。
- **操作映射 Operation Mapping**: 操作列表展示序号与快捷触发键位之间的对应关系，用于保证“所见即所得”的执行结果。
- **保存触发上下文 Save Trigger Context**: 执行 `s-enter` 时的上下文信息，包含条目类型、是否存在别名、目标粘贴位置与可写状态。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在 20 次“单条目 F2 别名操作”测试中，别名新增或更新成功率达到 100%。
- **SC-002**: 在 20 次“操作列表序号自动执行”测试中，`c-a-num` 触发结果与列表序号一致率达到 100%。
- **SC-003**: 在“单文件且有别名”样本测试中，使用 `s-enter` 后一次完成重命名并粘贴成功率不低于 95%。
- **SC-004**: 在“组合粘贴/纯文本”样本测试中，`s-enter` 触发后保持原组合粘贴行为的一致率达到 100%。

## Assumptions

- 本次范围仅覆盖单条目操作体验，不包含批量多条目同时执行的新增规则。
- 收藏页现有弹出层交互与视觉表现保持不变，仅补充别名逻辑能力。
- 目标粘贴位置与系统权限在正常可用状态下执行，异常权限处理遵循现有统一提示机制。
- 快捷键迁移仅针对“按序号自动执行操作”场景，不扩展到其他历史快捷键定义。
