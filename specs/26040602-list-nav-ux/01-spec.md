# 列表导航与多选视觉 — 需求

**基线**：2026-04-06，EzClipboard 主列表 `ClipItemList` / `useVirtualListScroll`。

## 问题

1. 键盘向上下导航时列表滚动过度，出现明显闪烁/抽动。
2. 键盘焦点高亮与多选勾选态在视觉上难以区分。

## 目标

1. 当前行已在滚动容器内完全可见且非强制滚动时，不因「居中偏好」再次滚动。
2. 单次导航避免无必要的多帧重复滚动（保留 DOM 未就绪时至多一次重试）。
3. 焦点行：主色左轨 + 轻焦点环；多选已勾选：独立勾选标记 + 中性底色，二者组合时可读。

## 非目标

- 不改 `public/`、`preload`、`listener`。
- 不改为 TanStack Virtual 真实接入（`virtualizer` 仍为 null 的路径保持可工作）。
