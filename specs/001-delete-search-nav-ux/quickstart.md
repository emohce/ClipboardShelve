# Quickstart: 手工验证 001-delete-search-nav-ux

**环境**: `pnpm install` 后 `pnpm run serve`，在 uTools 或浏览器宿主中按实际集成方式打开插件（与日常开发一致）。

## 1. 删除稳定性（SC-001）

1. 在普通列表选中一条，执行删除（与产品一致：Delete / Backspace / 菜单）。  
2. 确认该行消失；**不要**执行撤销（若产品无撤销则跳过）。  
3. 连续重复多次（≥20 次可在不同条目上），关闭再打开插件，确认数据无“静默复活”。  
4. 若曾遇“删了又出现”，打开开发者工具看是否仅在会话内出现；若是，优先核对持久化与防抖（见 `research.md` R-001）。

## 2. 搜索意外退出（SC-002）

1. 展开搜索，输入已知会触发的字符（从历史缺陷列表或逐步扩大 ASCII/符号/数字）。  
2. 观察：搜索面板是否保持、`filterText` 是否被无故清空、是否触发 `onEmpty` 等效行为。  
3. 记录：**按键 / 输入法状态 / 是否多选**。

## 3. 中文 IME + Enter（SC-003）

1. 焦点在搜索框，切换到拼音输入法，**不要确认**汉字，保持候选条。  
2. 按 **Enter** 确认候选。  
3. 期望：候选进入搜索框并参与筛选；**不应**执行复制当前列表项或关闭插件主流程。  
4. 无组字时按 Enter：记录当前产品设计（`list-enter`）；仅验证不与 IME 场景冲突。

## 4. 列表滚动可见（SC-004）

1. 准备长列表（超出视窗）。  
2. 仅使用 **ArrowUp / ArrowDown**（及必要时 PageUp/PageDown），从中间移动到 **第一条**、**最后一条**及随机行。  
3. 每次停稳后检查：高亮行是否**完整可见**；中间区域是否大致**垂直居中**（允许小偏差）。  
4. 快速连按方向键，确认最终停留行仍完整可见。

## 5. 回归嗅探

- 多选模式、收藏 Tab、锁定筛选 chip、`TagSearchModal` 打开时，重复关键用例，确认无新增副作用。

## 6. 顶栏与列表间距

1. 默认宽度与 **≤900px、≤720px** 下切换窗口，观察 **Tab 行 + 按钮行** 下方到 **第一条列表** 之间是否无重叠、无大块无效空白。  
2. 切换到 **收藏** 并展开子标签（双行顶栏），确认 `.clip-break--with-sub` 下仍无重叠。  
3. 若调整 [`Main.vue` 中 `.clip-break`](../../src/views/Main.vue) 高度，同步检查 **暂无记录** 占位高度是否合理。
