# Feature: 剪贴板列表键盘移动与选中展示优化

## 1. 目标

- **分页移动**：支持通过「长按或等价连续触发」上/下方向键进行**分页式**跳转，选中项在可视区内**居中**展示，并具备明确的**选中样式与标记**（与单步移动一致可辨）。
- **分页方式一致**：长按 repeat 分页与 **Ctrl+上/下**（`list-page-up` / `list-page-down`）在**步长、居中、选中反馈**上须**一致**，不得两套语义。
- **分页与居中复用**：分页路径应**复用**现有「分页并居中」相关逻辑（与逐项强制居中区分），避免重复实现或行为分叉。
- **单步移动**：单击上/下键时采用「**已完全可见则不滚动，否则滚至完全可见**」；**仅部分露出视口不算可见**。单步移动时选中项**始终可感知**。
- **删除后一致性**：删除当前项后，选中自动迁移到合理下一条（或空列表边界行为与现有一致），**不出现**选中错位、错误行高亮、DOM 与数据不同步或明显闪烁割裂。
- **锁/收藏状态**：锁、收藏等状态在列表行上**及时**反映，不依赖「先完成移动选中再刷新」才更新。
- **单行信息密度**：锁/收藏等图标与正文在同一行展示时，**不因图标换行**破坏布局；空间不足时对可缩略部分**省略或收缩**，保持单行稳定。
- **性能**：与单行渲染相关的抽取或调整**不得**明显拖慢长列表；**自动分页滚动**与**高亮选中**在一次分页交互中的**合计**耗时不应超过 **300ms**（以 `04-verify.md` 记录为准）。
- **筛选/搜索**：在搜索、筛选导致 `showList` 变化时，键盘移动与选中状态**不错乱**。
- **回归核对**：对近期与 list 展示相关的改动做**有依据**的核对（必要时参考公开实践），确保本次优化与之一致、不引入重复冲突。

- 原始需求见 [`docs/todo/260405-交互优化/移动效果优化/zt-move-原始需求.md`](../../../docs/todo/260405-交互优化/移动效果优化/zt-move-原始需求.md)。
- 列表键盘导航、分页、`setKeyboardActiveIndex` / `scrollActiveNodeIntoView`、虚拟列表滚动对齐等集中在 [`src/cpns/ClipItemList.vue`](../../../src/cpns/ClipItemList.vue)。
- 主窗口热键在 [`src/global/hotkeyBindings.js`](../../../src/global/hotkeyBindings.js) 中注册 `list-nav-up` / `list-nav-down`（单步）与 `list-page-up` / `list-page-down`（Ctrl+方向键分页）等 feature。
- 原始描述中「可键」按**笔误**理解为「**可见**」。已确认：**仅部分露出不算可见**，须**完全在列表可视区内**才适用「不滚动」。

## 3. 用户场景 / 使用场景

- **场景 A（分页 + 居中）**：用户按住上/下键触发连续分页（或产品定义的等价操作），每次有效跳转后当前条目在列表可视区**居中**，且高亮与角标清晰。
- **场景 B（单步·已完全可见）**：当前选中项**完全**在列表可视区内（非仅部分露出），用户单击上/下键切换相邻项，列表**不无故滚动**。
- **场景 C（单步·未完全可见）**：当前项仅部分露出或完全不可见，用户单击上/下键后，新选中项**滚至完全可见**，高亮完整可辨。
- **场景 D（删除）**：用户删除当前高亮项，选中自动落到下一条（或列表末尾上一条），行内容与持久化数据一致，无错行高亮、无整列表无意义闪烁。
- **场景 E（锁/收藏）**：用户切换锁或收藏后，对应行图标与状态**立即**与数据一致，且不引起行内异常换行。
- **场景 F（搜索后导航）**：用户输入筛选条件缩小列表后，用方向键移动，索引与展示列表一致，无越界或幽灵选中。

## 4. 非目标

- 不改变 uTools 插件入口契约、[`public/plugin.json`](../../../public/plugin.json) / preload 等与本列表交互无直接关系的全局配置，除非验证证明必须联动修改。
- 不将本次扩展为「重做虚拟列表库」或引入新的列表滚动框架。
- 不强制在本次修改「设置页」热键文案或默认键位（若与行为冲突，在实现阶段于 plan 中说明是否仅文档同步）。
- 「联网检索」作为实现阶段的参考手段，不在 spec 中规定具体检索关键词或工具。

## 5. 验收标准

- **分页**：分页移动后，活动项在可视区内为**居中**对齐（与现有 `block: "center"` 等行为一致，允许与虚拟列表实现对齐的像素级偏差），且选中样式与标记可见。
- **分页一致**：**Ctrl+上/下** 与 **长按 repeat 分页** 行为一致（步长、居中、选中反馈无差异）。
- **分页复用**：分页路径与现有 `list-page-up` / `list-page-down`（或统一封装的滚动/索引函数）**共享**同一套「目标索引 + 滚动对齐」策略，无两套互斥的居中逻辑。
- **单步**：若移动后活动项**完全**在可视区内（**部分露出不算**），则不产生额外滚动；否则滚动至**完全可见**，高亮正确。
- **删除**：删除当前项后 `activeIndex` 与 `showList`、持久化数据一致；无错误行 `active`、无列表项 key 错乱导致的渲染错位。
- **锁/收藏**：状态变更后，对应行在**下一帧或同等及时性**内更新，不依赖再次方向键导航才正确。
- **布局**：典型窗口宽度下，锁/收藏图标与标题同列不换行；极端窄窗下降级为省略/收缩，不挤爆布局。
- **性能**：长列表场景下方向键与删除操作无明显可感知卡顿；**自动分页**时「滚动 + 高亮选中」**合计**不超过 **300ms**（度量方式与环境以 `04-verify.md` 为准，未测不得写已通过）。
- **筛选**：筛选/搜索前后，键盘导航索引与列表数据一致。

## 6. 边界与异常

- 列表为空、仅一项、在首尾分页：行为符合现有边界（不越界、不崩溃）。
- 虚拟列表快速滚动与键盘并发：选中项仍与 DOM 节点对应，无空白高亮。
- 多选模式（若与单选 active 并存）：不破坏现有多选语义；冲突时在 plan 中写明优先级。
- 加载更多（列表尾部）：分页/单步与加载触发条件不互相死锁。
- 「完全可见」与虚拟列表边界的判定若存在亚像素差异，在 `04-verify.md` 中记录可接受误差。

## 8. 已确认项

1. **分页方式**：**Ctrl+上/下** 与 **长按方向键（repeat）** 触发的分页在**行为上一致**（步长、居中、选中反馈相同）。
2. **「可见」**：单步「可见则不滚」中，**部分露出视口不算可见**；仅当条目**完全**在列表可视区内时跳过滚动。
3. **300ms**：指单次**自动分页**交互中，**分页滚动**与**高亮选中**效果的**合计**耗时不应超过 300ms；具体度量在 `02-plan.md` / `04-verify.md` 中约定。
