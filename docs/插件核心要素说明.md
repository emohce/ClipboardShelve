# EzClipboard 插件核心要素说明（列表）

> 本文用**列表**归纳 uTools 插件在本仓库中的必备契约与路径，便于对照构建、调试与发版。细节实现见源码与 [README.md](../README.md)。

---

## 1. 宿主入口与元数据

- **插件清单**：由 [`scripts/utools-runtime-assets.mjs`](../scripts/utools-runtime-assets.mjs#L4) 生成到 [`dist/plugin.json`](../dist/plugin.json)
  - **生产主入口**：`main` → `index.html`（加载 [`dist/index.html`](../dist/index.html)，由构建生成）
  - **预加载脚本**：`preload` → `preload.js`（与清单同目录，构建后位于 `dist/preload.js`）
  - **开发调试页**：`development.main` → `http://localhost:8081/`（须与 [`vite.config.js`](../vite.config.js) 中 `server.port` 一致，当前为 **8081**）
  - **图标**：`logo` → `logo.png`
  - **平台**：`platform`（如 `win32` / `darwin` / `linux`）
  - **功能与关键词**：`features`（`code`、`explain`、`cmds` 等，供 uTools 搜索与展示）

---

## 2. 预加载与全局桥接（`preload.js`）

- **文件**：由 [`scripts/utools-runtime-assets.mjs`](../scripts/utools-runtime-assets.mjs#L30) 生成到 [`dist/preload.js`](../dist/preload.js)
- **作用**：在渲染进程挂载 **`window.exports`**，供页面与业务层使用
- **当前注入能力（摘）**：
  - `utools`
  - `path`、`os`、`sep`
  - `existsSync`、`readFileSync`、`writeFileSync`、`copyFileSync`、`mkdirSync`、`watch`、`rmSync`、`unlinkSync`、`rmdirSync`
  - `crypto`、`Buffer`
  - `clipboard`、`nativeImage`（Electron）
  - `listener`（见下节）
  - `time`（Worker 包装的时间工具，构建后位于 `dist/time.js` / `dist/time.worker.js`）

---

## 3. 剪贴板监听子进程（`listener.js`）

- **文件**：由 [`scripts/utools-runtime-assets.mjs`](../scripts/utools-runtime-assets.mjs#L68) 生成到 [`dist/listener.js`](../dist/listener.js)
- **作用**：按平台启动本机剪贴板监听可执行文件，通过事件通知前端（如 `change`）；与 `initPlugin`、轮询降级等共同构成监听链路

---

## 4. 前端应用入口与界面

- **HTML 入口**：根目录 [`index.html`](../index.html)（开发由 Vite 提供；生产打包进 `dist`）
- **脚本入口**：[`src/main.js`](../src/main.js)
  - 先 **`initPlugin()`**（[`src/global/initPlugin.js`](../src/global/initPlugin.js)）
  - 再 **`ensureDevDbStub()`**（[`src/global/devDbStub.js`](../src/global/devDbStub.js)，浏览器/未就绪时兜底）
  - 再挂载 Vue 根组件 [`App.vue`](../src/App.vue)
- **页面与组件**：[`src/views/`](../src/views/)、[`src/cpns/`](../src/cpns/)；全局能力见 [`src/global/`](../src/global/)

---

## 5. 运行时全局对象（业务侧约定）

- **`window.exports`**：由 preload 注入（见 §2）
- **`window.db` / `window.remove` / `window.listener`**：在 [`initPlugin`](../src/global/initPlugin.js) 初始化过程中挂到 `window`，供列表、设置、搜索等使用（与纯浏览器 SPA 不同）

---

## 6. 构建与产物目录

- **开发**：`pnpm run serve` → 先执行 `scripts/prepare-utools-runtime.mjs` 生成运行时资产，再启动本机 `http://localhost:8081/`，与 `plugin.json` 的 `development.main` 对齐
- **生产构建**：`pnpm run build`（或 [`build.sh`](../build.sh)）→ 输出目录 **`dist/`**
- **Vite 配置要点**：[`vite.config.js`](../vite.config.js)
  - `base: './'`：相对路径，适配插件内打开 `index.html`
  - `publicDir: false`：不把整个 `public` 当 Vite 默认静态目录；**uTools 静态入口由自定义插件在 build 结束时生成**
- **构建或开发准备时生成到 `dist/` 根目录的文件**（由 `scripts/utools-runtime-assets.mjs` 生成）：
  - `plugin.json`（插件清单）
  - `preload.js`（预加载脚本）
  - `listener.js`（剪贴板监听）
  - `logo.png`（插件图标）
  - `time.js` / `time.worker.js`（时间工具 Worker）
- **发版**：对 **`dist/` 内全部内容**打 zip，扩展名改为 `.upx`（或按 uTools 后台要求），上传市场；本地也可用 uTools 开发者工具直接加载 **`dist/`** 目录验证
