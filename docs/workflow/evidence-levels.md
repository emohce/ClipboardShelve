# Evidence Levels

## 目标
- 统一判断依据的可信度，避免把猜测、静态阅读和真实运行结果混为一谈。

## E1 Code Reading
- 来源：静态代码阅读、调用链分析、结构推断。
- 适用：说明“代码看起来会怎样”。
- 不适用：直接当作运行时结论。

## E2 Local Command Output
- 来源：本地命令输出、日志输出、文本结果。
- 适用：证明某条命令、某段日志、某个文件内容实际存在。
- 注意：不等同于真实交互行为已验证。

## E3 User UI Feedback
- 来源：用户在界面中的实际观察与反馈。
- 适用：证明某个交互路径在用户环境下的表象结果。
- 注意：仍需区分表象与真实根因。

## E4 Runtime Confirmation
- 来源：在目标运行时中的真实确认，例如 `uTools` 插件环境、浏览器交互、桌面窗口实际行为。
- 适用：确认真实通路、真实焦点、真实滚动、真实事件链。
- 优先级最高。

## 使用规则
- 关键判断后尽量标明证据等级，例如“判断：滚动命中了真实祖先链，证据 `E3 + E4`”。
- 只有 `E1` 时，不把推断写成“已确认”。
- `E3` 说明用户观察结果，`E4` 说明真实运行通路确认；两者不能混用。
