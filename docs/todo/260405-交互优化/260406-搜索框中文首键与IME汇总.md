# 搜索框「键入展开」中文输入法首键问题 — 汇总

**基线**：EzClipboard 主界面，搜索条未展开时，用户希望在列表区直接按键用拼音/中文搜索。  
**状态**：已用组合手段缓解；偏实现向取巧，但在 Chromium / uTools 场景下可用。  
**相关代码**：`src/views/Main.vue`、`src/cpns/ClipSearch.vue`。

---

## 1. 现象

- 未展开搜索时，从列表直接输入中文（如打拼音选「解决」），框内易出现 **`拉丁首字母 + 汉字`**，例如 `j解决`。
- 期望：只有 **`解决`**（或完整拼音/选字结果），不要多一个误入的 `j`。

---

## 2. 根因（简述）

历史上存在两条叠加问题：

| 层级 | 说明 |
|------|------|
| 焦点时机 | 若在 `document` **冒泡**阶段才把焦点交给 `.clip-search-input`，本次按键的默认输入 / `beforeinput` 往往仍关联**原焦点**（如列表或 `.main`），不利于 IME 把首键交给搜索框。 |
| 浏览器行为 | 即使用**捕获阶段**提前 `focus()`，部分环境下仍会对**第一个物理键**先走一次普通的 `insertText`（拉丁字母），随后 IME 再合成中文，视觉上仍是「字母抢先插入」。 |

---

## 3. 当前方案（两层 + 一层补救）

### 3.1 `Main.vue`：捕获阶段展开并聚焦

- 使用 `document.addEventListener('keydown', handler, **true**)`，在事件的 **capture** 阶段（早于原 `target`）：
  - 展开搜索区（必要时去掉 `.clip-search` 的 `display: none`）。
  - `window.focus?.()` 后 `input.focus()`。
- **不做** `preventDefault()`，避免打断输入法。
- 守卫：有其它热键层（`getCurrentLayer()`）、或多选、或焦点已在非搜索的 `input/textarea/contenteditable` 时不抢键。
- 从列表「键入展开」时同时写入：
  - `revealFromKeyAt`：时间戳（`Date.now()`）。
  - `revealOpeningKey`：本次 `keydown` 的 `key`（供子组件只做**有针对性的**拦截）。

### 3.2 `ClipSearch.vue`：吃掉误入的拉丁首字符（取巧但有效）

仅在 **`revealOpeningKey` 为单个英文字母** 且时间窗内（实现里约 800ms）生效：

1. **`beforeinput`**：若 `inputType === 'insertText'`、`!isComposing`、`data` 为**单个** `[a-zA-Z]` 且与 `revealOpeningKey` 大小写无关地一致 → **`preventDefault()`**，并 `emit('revealKeyGuardUsed')` 通知父级清掉 guard。
2. **`compositionstart` 兜底**：若框内仍只剩**一个**与开局键相同的拉丁字母 → 清空本地 `filterText` 并同步父级，同样 `revealKeyGuardUsed`；并用 **`suppressNextEmptyPanel`** 避免误触「删空即关搜索」的 `onEmpty`。

父组件在 `reveal-key-guard-used`、清空搜索、全量 `resetPluginUiState` 时调用 **`clearSearchRevealKeyGuard`**。

### 3.3 刻意不覆盖的场景

- 点「搜索」、**Ctrl+F** 等展开：**不**设置 `revealFromKeyAt` / `revealOpeningKey`，不走过 guard。
- 用**数字或其它符号**从列表展开开局：**不**按「拉丁首键」规则拦截（`revealOpeningKey` 非单字母时子组件逻辑直接不生效）。

---

## 4. 取舍与风险

| 项目 | 说明 |
|------|------|
| 取巧性 | 依赖 `beforeinput` + 与开局键相同的单次 `insertText` 假设；不同输入法事件顺序若异常，可能需再调时间窗或补规则。 |
| 英文检索 | 用户若**用同一字母键**从列表展开并只想输入英文单词，**首字母可能被拦一次**，需再敲一次该字母，或改用按钮/Ctrl+F 先聚焦搜索框。 |
| 维护注意 | 若新增其它「从外链入展开搜索」的路径，需统一是否设置 / 清理 `revealFromKeyAt`、`revealOpeningKey`。 |

---

## 5. 验证建议（手工）

1. 搜索未展开，中文输入法在列表区直接输入，框内应为纯中文/拼音流程结果，**无**孤立前缀拉丁字母（如不再出现 `j解决`）。
2. Ctrl+F 展开、点击「搜索」展开：行为与 guard 无关，不应异常关面板。
3. 清空搜索或退出插件状态重置后，再次键入展开仍正常。

构建：仓库默认 `pnpm run build`；uTools 内输入法需在**真实插件窗口**下各验一轮。

---

## 6. 非目标

- 不保证所有第三方输入法、所有桌面环境的完全一致事件序。
- 不改变热键注册表与 `public/` 运行时脚本行为；本改动仅限上述 Vue 页面与搜索子组件。
