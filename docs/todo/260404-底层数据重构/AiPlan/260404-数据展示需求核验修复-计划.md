### 2. 缓存与 `utools.db` 的映射关系

[src/global/utoolsDB.js](../src/global/utoolsDB.js)：

- **内存**：`this.dataCache` 持有 `data`、`collects`、`collectData`、`tags`、`tagUsage` 与时间戳。
- **持久化**：`utools.db` 文档 `_id` 前缀 `clipboard:` — `clipboard:index`、`clipboard:data:<id>`、`clipboard:collects`、`clipboard:tags`；单条历史为 **一文档一条**，`data` 字段为完整 item 对象（含图片 base64、`thumbnail` 等）。
- **兼容层**：`get dataBase()` 返回 `this.dataCache`，与旧 `window.db.dataBase` 访问方式对齐。
- **注意**：`bulkPutHistoryItems`（迁移）按输入数组顺序重写 `dataCache.data`，依赖传入 JSON 顺序；正常运行时 `addItem` / `_sortData` 按 `updateTime` 排序。初始化 `_loadAllDataItems` 按 `updateTime` 排序加载。

### 3. uTools 存储与迁移正确性

| 项 | 结论 |
|----|------|
| **模式选择** | [src/global/initPlugin.js](../src/global/initPlugin.js)（约 L1053-1084）：`storageMode` 存 `utools.dbStorage`；新用户无 JSON → 默认 `utools`；存在 JSON → 默认 `json`，需用户设置或迁移后切 `utools`。逻辑自洽。 |
| **迁移流程** | [src/global/dbMigration.js](../src/global/dbMigration.js) `executeMigration`：备份 JSON → 校验 → `wipeAllClipboardDocs` → `bulkPutHistoryItems` + 收藏/标签写入；**不**先 `init()` 避免大库 `allDocs` 双次加载，合理。 |
| **回滚缺陷** | [src/views/Setting.vue](../src/views/Setting.vue) `handleRollbackMigration`（约 L971-975）使用 `dbPath + '.backup.' + Date.now()` 作为 `backupPath`，**与迁移时真实备份文件名几乎不可能一致**，回滚会从错误路径拷贝，**迁移回滚当前不可靠**（todo `migration-rollback-backup-path`）。 |
| **异步写入** | `UToolsDB.addItem` / `removeItem` 等为 **async**，[src/global/initPlugin.js](../src/global/initPlugin.js) `handleClipboardChange` 调用 `db.addItem(item)` **未 await**，极端情况下可能与快速连续剪贴事件竞态（todo `utools-async-additem-review`）。 |

## 用户反馈与截图结论（优先）

参考：Cursor 工作区截图（文件 tab、单条文件项、右侧操作图标已出但整卡对比弱）。

### 1. 最末项 ↓ 键无法露出下一项（Bug）

**现象**：在已加载列表的最后一项继续按 ↓，后续条目不出现或未滚入视口。

**可能原因（实现时逐项排除）**：

- **懒加载时序**：[src/cpns/ClipItemList.vue](../src/cpns/ClipItemList.vue) 仅在 `activeIndex >= length - THRESHOLD` 时 `emit('loadMore')`；若 **`list-nav-down` 在已是最后一项时直接 return** 且不先 `loadMore`，则永远不会追加数据。需核对 `registerFeature('list-nav-down')` 是否在 `activeIndex === length - 1` 时仍触发父级加载再 `setKeyboardActiveIndex`。
- **DynamicScroller**：`scrollToItem` 在新行插入前调用会无效；应在 **`showList` 更新后 `nextTick`（或双帧）再 `scrollToItem(newIndex)`**，必要时传对齐参数（以库 API 为准）。
- **过滤后无下一项**：与「真无数据」区分，避免无效重试。

**验收**：末项 ↓ 后，若主列表仍有未加载段，必须先加载再选中并滚入视口；若无更多项则保持末项高亮且不报错。

### 2. 选中缺少「框选」、对比不足（UX）

**现象**：`ClipOperate` 显示即表示 active，但卡片与浅蓝容器背景 **边界不清**。

**方向**（择一或组合，保持与主题变量一致）：

- `outline: 2px solid @primary-color` 或 `box-shadow: 0 0 0 2px ...`；
- 选中卡 **略深实底** + **主色左边条**（与多选 `::before` 区分粗细/色阶）；
- 避免仅靠 `z-index`，确保 **浅色主题下 Δ 对比** 足够。

**验收**：不依赖鼠标悬停即可从截图距离辨认当前选中行。

### 3. 图片式层叠：默认单行缩略，仅选中展开（UX + 性能）

**目标**：与需求「层叠 + 第一行信息」一致——**默认**每条仅 **一行**（时间/类型图标 + 单行 `ellipsis` 摘要），视觉上图卡 **层叠缩略**；**选中** 再展开全文、大图、文件列表等。

**实现要点**：

- [src/cpns/ClipItemRow.vue](../src/cpns/ClipItemRow.vue)：用 `isActive` 控制 `v-show`/`class` 切换「摘要行」与「详情块」；非选中隐藏 `FileList` 大段、图片可缩为小 thumb 或仅图标。
- [src/style/cpns/clip-item-list.less](../src/style/cpns/clip-item-list.less)：非选中 `.clip-data` **单行** `white-space: nowrap; overflow: hidden; text-overflow: ellipsis; max-height: 1.5em`（或固定行高）；选中移除限制并保留过渡。
- **DynamicScroller**：折叠态 **`min-item-size` 可显著减小**（与单行高一致）；`size-dependencies` 已含 `activeIndex === index`，确保切换 active 时 **行高重算**；若库需显式调用父组件 `scrollerRef.updateSize(index)`，在 `watch(activeIndex)` 或 row emit 中补上。
- 层叠外观：略 **负 margin 或 translateY** 叠压感（注意不与无障碍焦点环冲突）。

**验收**：默认列表信息密度接近「缩略条」；选中后展开与当前截图信息量相当或更完整，且滚动/键盘仍正常。
