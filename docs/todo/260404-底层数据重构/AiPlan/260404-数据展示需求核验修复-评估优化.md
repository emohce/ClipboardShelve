## 二、联网检索关键结论

### 1. 虚拟滚动选型

| 方案 | 优势 | 劣势 | 适用性 |
|------|------|------|--------|
| **vue-virtual-scroller DynamicScroller** (已有依赖) | 零新依赖；DynamicScrollerItem 自动测量；社区成熟 | `scrollToItem` 在不等高场景已知有偏移 bug（GitHub #696/#307），需双次 nextTick 修正；最后更新 2022 年 | **推荐主路径**——已在项目中，切换成本最低 |
| **@tanstack/vue-virtual** | 框架无关、headless、活跃维护、10-15kb | 动态高度向上滚动时有 stutter（Issue #659）；需自行写 Vue 3 适配模板 | **备选路径**——若 DynamicScroller 实测不可用时迁移 |
| **virtua** | 零配置、极低开销、支持动态高度 | Vue 适配不成熟；社区小 | **暂不推荐** |

**结论**：继续 `vue-virtual-scroller`，从 `RecycleScroller` 升级为 `DynamicScroller`；`scrollToItem` 偏移用「nextTick → scrollToItem → nextTick → scrollToItem」双次修正。不新增依赖。

### 2. 选中无透明叠压——最高效实现

联网调研与 CSS stacking context 最佳实践对比：

| 方案 | 原理 | 工作量 | 推荐 |
|------|------|--------|------|
| **A: DynamicScroller 高度更新 + 不透明背景** | 选中行展开 → DynamicScrollerItem `updateSize` → 下方自然下移；背景 100% 不透明 | 中（需处理 updateSize 时机） | ✅ **首选** |
| **B: z-index + 不透明覆盖** | 选中行 z-index 提高 + solid background 盖住下方 | 低 | ✅ 作为 A 的补充保障 |
| **C: Teleport 浮层** | 选中行内容 Teleport 到 body 级浮层 | 高（双模板） | ❌ 维护成本过高 |

**结论**：采用 **A+B 组合**——DynamicScroller 自然推开 + `.clip-item.active` 强制 `background: @bg-elevated-color`（100% 不透明）+ `z-index: 10`。最少代码改动，零新依赖。

### 3. 键盘长按——最高效实现

- **主方案**：按键区分放行 `e.repeat`（仅 Arrow/Page 键）——改 `hotkeyRegistry.js` dispatch 函数约 5 行
- **兜底不必要**：现代 Chromium（uTools WebView）的 keydown repeat 行为一致（~33ms 间隔）；rAF 兜底增加复杂度但收益极低
- **优化**：结合 `scrollToItem` 时使用 `requestAnimationFrame` 合并连续滚动请求，避免每次 repeat 都触发布局

## 三、优化后的实施计划（按优先级排序）

### Phase 1：核心交互修复（阻塞验收）

#### 1.1 热键 repeat 放行（~5 行改动）

- **文件**：[src/global/hotkeyRegistry.js](../src/global/hotkeyRegistry.js#L143-L146)
- **改法**：dispatch 函数中，`ignoreRepeat && e.repeat` 早退前，检查 `e.key` 是否为 `ArrowUp|ArrowDown|PageUp|PageDown`，若是则不早退
- **预估**：5 行内改动

```javascript
// 改前：
if (ignoreRepeat && e.repeat) return false;
// 改后：
const REPEAT_ALLOWED_KEYS = new Set(['ArrowUp', 'ArrowDown', 'PageUp', 'PageDown']);
if (ignoreRepeat && e.repeat && !REPEAT_ALLOWED_KEYS.has(e.key)) return false;
```

#### 1.2 物理 PageUp/PageDown 绑定（+2 行）

- **文件**：[src/global/hotkeyBindings.js](../src/global/hotkeyBindings.js#L246-L249)
- **改法**：在 `ctrl+ArrowUp/Down` 绑定旁增加：

```javascript
{ layer: "main", shortcutId: "PageUp", features: ["list-page-up"] },
{ layer: "main", shortcutId: "PageDown", features: ["list-page-down"] },
```

#### 1.3 移除 Home/End 绑定（-2 行 + 清理）

- **文件**：[src/global/hotkeyBindings.js](../src/global/hotkeyBindings.js#L248-L249) 删除 Home/End 行
- **文件**：[src/cpns/ClipItemList.vue](../src/cpns/ClipItemList.vue#L1334-L1348) 删除或置空 `list-home`/`list-end` feature

### Phase 2：视觉与布局（阻塞验收）

#### 2.1 时间独占第一行

- **文件**：[src/style/cpns/clip-item-list.less](../src/style/cpns/clip-item-list.less#L126-L129)
- **改法**：`.clip-info` 改为 `flex-direction: column`；`.clip-time` 占满宽度作为第一行；`.clip-data` 作为第二行
- **配合**：`ClipItemRow.vue` 模板结构微调（时间行独立一行、摘要独立一行）
- **效果**：层叠时首行露出「时间 + 第一行内容摘要」

#### 2.2 选中态：无透明叠压

- **文件**：[src/style/cpns/clip-item-list.less](../src/style/cpns/clip-item-list.less#L44-L67)
- **改法**：
  - `.clip-item` 默认背景改为 `background-color: @bg-elevated-color`（100% 不透明，去掉 `fade(..., 88%)`）
  - `.clip-item.active` 增加 `z-index: 10; background-color: @bg-elevated-color;`（确保不透明）
  - 选中行展开时若高度变化，DynamicScroller 自动推开下方行
- **验收标准**：选中行完整信息可读，下方行内容不从当前行透出

### Phase 3：虚拟化升级（阻塞验收）

#### 3.1 RecycleScroller → DynamicScroller

- **文件**：[src/cpns/ClipItemList.vue](../src/cpns/ClipItemList.vue#L8-L16)
- **改法**：

```vue
<!-- 改前 -->
<RecycleScroller :items="showList" :item-size="60" ... />

<!-- 改后 -->
<DynamicScroller :items="showList" :min-item-size="54" key-field="id" ...>
  <template v-slot="{ item, index, active }">
    <DynamicScrollerItem :item="item" :active="active"
      :size-dependencies="[activeIndex === index, item.type, item.id]"
      :data-index="index">
      <ClipItemRow ... />
    </DynamicScrollerItem>
  </template>
</DynamicScroller>
```

- **import 修改**：`RecycleScroller` → `DynamicScroller, DynamicScrollerItem`

#### 3.2 scrollToItem 双次修正

- **文件**：[src/cpns/ClipItemList.vue](../src/cpns/ClipItemList.vue#L1163-L1177)
- **改法**：

```javascript
const scrollActiveNodeIntoView = (index = activeIndex.value) => {
  const doScroll = () => {
    if (scrollerRef.value?.scrollToItem) {
      scrollerRef.value.scrollToItem(index);
    }
  };
  // 双次 nextTick：第一次触发测量，第二次修正偏移（vue-virtual-scroller #696 workaround）
  nextTick(() => { doScroll(); nextTick(doScroll); });
};
```

#### 3.3 getPageStep 动态化

- **文件**：[src/cpns/ClipItemList.vue](../src/cpns/ClipItemList.vue#L1190-L1200)
- **改法**：`min-item-size` 替代硬编码 `60`；容器高度用 `scrollerRef.value?.$el?.clientHeight`
- **resize 监听**：`onMounted` 中加 `ResizeObserver` 监听列表容器，`getPageStep` 已引用 `clientHeight` 无需额外缓存

### Phase 4：边界与 a11y（低成本必做）

#### 4.1 空列表 / 加载中

- 空列表：`showList.length === 0` 时显示「暂无数据」占位（已有类似逻辑可复用）
- 加载中：若异步有延迟，加 `v-if` 文案切换

#### 4.2 轻量 a11y

- 列表容器：`role="listbox"`
- 行：`role="option"` + `:aria-selected="isActive"`
- 成本极低，约 3 行模板属性

### Phase 5：可选 Polish（不阻塞）

- 骨架屏、交错淡入、波纹、玻璃拟态
- rAF 长按兜底（实测 Chromium repeat 若有问题再加）
- `utools.setExpendHeight` 窗口高度调整

## 四、与原计划的差异摘要

| 维度 | 原计划 | 优化后 | 理由 |
|------|--------|--------|------|
| key-repeat-fallback | pending（可选） | **降为 Phase 5** | Chromium WebView repeat 行为一致，rAF 兜底收益/成本比低 |
| 虚拟化选型 | DynamicScroller + 降级路径 2（Teleport 覆盖层） | **仅保留 DynamicScroller 单路径** | Teleport 覆盖层维护成本高、双模板不合理；DynamicScroller + 不透明背景已满足「无透明叠压」 |
| 选中无透明叠压 | A 下移 / B 置顶不透明 / Teleport 三选一 | **A+B 组合确定** | DynamicScroller 自然推开 + 不透明背景是最低成本方案 |
| scrollToItem 偏移 | 未提及具体 workaround | **双次 nextTick 修正**（联网确认 #696） | vue-virtual-scroller 已知 bug，不修会导致高亮项滚不到位 |
| 运营侧栏重命名 | pending | **移出本迭代** | 不属于「数据展示」核心需求 |
| 阈值 500 切换 | 可选始终虚拟化 | **始终 DynamicScroller** | 避免双实现分支；DynamicScroller 在小数据量下开销可忽略 |
| 动效库 | 不默认引入 @vueuse/motion | **维持不引入** | CSS transition 足够 |

## 五、风险与注意

1. **DynamicScroller `min-item-size` 选值**：需接近折叠态真实高度（预计 54-60px），过小会导致初次渲染多余测量、过大会导致空白闪烁
2. **`size-dependencies` 须完整**：`activeIndex` 变化时必须触发重算，否则选中展开后下方行不推开
3. **`scrollToItem` 双次修正在极端快速键盘操作下可能累积 nextTick**：建议用 `requestAnimationFrame` 合并——同一帧内多次调用只执行最后一次
4. **样式 100% 不透明可能影响 few-items 模式下的视觉层次感**：需人工验收微调阴影补偿
