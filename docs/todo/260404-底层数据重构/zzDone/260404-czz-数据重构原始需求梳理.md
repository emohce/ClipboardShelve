## 调研与建议（Sweep调研补充）

### 需求完整性核验

基于对两个项目的代码分析和行业最佳实践调研，当前需求覆盖了核心问题，但建议补充以下方向：

**建议补充的需求点：**
- **数据同步与冲突解决策略**：uTools支持多设备同步，需要明确数据冲突时的处理机制
- **数据迁移与版本管理**：当前数据结构变更时的迁移方案
- **性能监控与指标**：建立性能基准，量化优化效果（如首屏加载时间、列表滚动FPS、批量操作耗时）
- **可访问性（A11y）**：键盘导航、屏幕阅读器支持
- **国际化（i18n）**：多语言支持架构预留

### 技术调研与最佳实践

#### 1. 列表渲染性能优化

**当前状态对比：**
- **ClipboardManager**：使用原生 `v-for` 直接渲染所有数据，存在性能瓶颈
- **ClipboardShelve**：已集成 `vue-virtual-scroller`，实现虚拟滚动，性能优于 ClipboardManager

**调研发现：**
- `vue-virtual-scroller` 是 Vue 3 生态中最成熟的虚拟滚动方案
- 核心原理：仅渲染可视区域及缓冲区的元素（通常几十条），DOM节点减少几个数量级
- 支持定高和不定高模式，ClipboardShelve 使用定高模式（`:item-size="60"`）

**进一步优化建议：**
1. **动态高度优化**：对于内容高度差异大的场景（如长文本预览），考虑使用 `DynamicScroller` 替代 `RecycleScroller`
2. **Buffer优化**：调整 `buffer` 参数平衡滚动流畅度和内存占用
3. **图片懒加载**：结合虚拟滚动，实现图片按需加载和缩略图策略
4. **v-memo优化**：对于静态内容使用 Vue 3 的 `v-memo` 指令减少不必要的重新渲染

**炫酷效果建议：**
- **骨架屏加载**：在数据加载时显示占位动画
- **交错动画**：列表项进入时的渐变动画
- **平滑滚动**：使用 CSS `scroll-behavior: smooth` 和虚拟滚动结合
- **拖拽排序**：集成 `vuedraggable` 实现拖拽重排

#### 2. 数据存储优化

**uTools 官方存储方案调研：**

uTools 提供三层存储 API：

1. **utools.db（NoSQL数据库）**
   - 基于文档模型，支持复杂查询
   - 支持批量操作：`bulkDocs(docs)`
   - 支持附件：`postAttachment` / `getAttachment`
   - 支持云同步：`replicateStateFromCloud()`
   - **警告**：多设备同时编辑同一文档会产生冲突，应将数据分散到多个文档

2. **utools.dbStorage（键值对存储）**
   - 基于上述数据库封装的简化 API
   - 适合配置、用户偏好等简单数据
   - API：`setItem(key, value)` / `getItem(key)` / `removeItem(key)`

3. **utools.dbCryptoStorage（加密存储）**
   - 加密版本的键值对存储
   - 适合敏感数据

**IndexedDB 最佳实践（参考）：**
- 异步操作不阻塞主线程
- 支持事务保证数据一致性
- 储存空间大（≥250MB）
- 支持索引加速查询

**当前问题分析：**
- ClipboardShelve 已使用 uTools db API，但可能存在数据结构不够分散的问题
- 批量操作卡死可能是因为事务处理不当或一次性处理过多数据

**优化建议：**
1. **数据分片策略**：按时间或类型将数据分散到多个文档，避免单文档过大
2. **索引优化**：为常用查询字段（如时间、类型、标签）建立索引
3. **批量操作分块**：将大批量操作拆分为小批次（如每批100条），配合 `requestIdleCallback` 或 `setTimeout` 分片执行
4. **缓存策略**：使用内存缓存热点数据，减少数据库访问
5. **预加载与懒加载结合**：初始加载少量数据，滚动时按需加载

**炫酷效果建议：**
- **数据可视化**：存储空间使用情况图表
- **云同步状态指示**：实时显示同步状态和冲突提示
- **数据导出/导入**：支持 JSON/CSV 格式备份

#### 3. 键盘快捷键系统重构

**当前问题：**
- ClipboardShelve 的快捷键逻辑分散在组件中（如 `ClipItemList.vue` 的 `keyDownCallBack`）
- 硬编码快捷键，不支持自定义
- 没有统一的 action 管理机制

**调研方案：KeyBindingManager 模式**

参考图形编辑器的快捷键管理设计，建议实现：

```javascript
interface IKeyBinding {
  key: IKey | IKey[];           // 快捷键定义
  winKey?: IKey | IKey[];       // Windows 特供版
  when?: (ctx: IWhenCtx) => boolean;  // 上下文判断
  action: (e: KeyboardEvent) => void; // 执行的动作
}

interface IKey {
  ctrlKey?: boolean;
  shiftKey?: boolean;
  altKey?: boolean;
  metaKey?: boolean;
  keyCode: string;  // KeyboardEvent['code'] 或 '*'
}
```

**核心特性：**
1. **多快捷键绑定**：同一 action 可绑定多个快捷键（如 Delete/Backspace 都删除）
2. **跨平台适配**：自动适配 Windows/macOS 快捷键差异
3. **上下文控制**：通过 `when` 函数决定是否执行
4. **短路匹配**：只执行第一个匹配的快捷键，避免冲突
5. **优先级管理**：支持高优先级快捷键覆盖默认绑定
6. **动态注册/注销**：运行时修改快捷键配置

**实现架构：**
```
HotkeyRegistry（全局注册中心）
  ├── ActionManager（动作管理）
  │   ├── Action 定义（独立逻辑）
  │   └── Action 执行器
  ├── KeyBindingManager（快捷键绑定）
  │   ├── 绑定注册
  │   ├── 优先级管理
  │   └── 上下文判断
  └── ConfigManager（配置持久化）
      ├── 快捷键配置
      └── 用户自定义
```

**炫酷效果建议：**
- **快捷键可视化编辑器**：图形化界面配置快捷键
- **冲突检测**：实时检测快捷键冲突并提示
- **快捷键提示**：按住快捷键显示可用操作列表
- **录制模式**：用户按下按键自动生成配置

#### 4. 批量操作优化

**当前问题：**
- 批量操作导致页面卡死，可能是因为同步执行大量 DOM 操作或数据库操作

**优化方案：**
1. **Web Worker**：将耗时计算放到 Web Worker 中执行
2. **时间分片**：使用 `requestIdleCallback` 或 `setTimeout` 分片处理
3. **进度反馈**：显示操作进度，提升用户体验
4. **取消机制**：支持中途取消批量操作
5. **事务批处理**：利用 uTools db 的 `bulkDocs` API 批量提交

**示例代码（时间分片）：**
```javascript
async function processBatch(items, batchSize = 100) {
  for (let i = 0; i < items.length; i += batchSize) {
    const batch = items.slice(i, i + batchSize);
    await processChunk(batch);
    // 让出主线程
    await new Promise(resolve => setTimeout(resolve, 0));
  }
}
```

#### 5. 交互效率提升

**调研发现：**
- Vue 3 性能优化十大技巧包括：懒加载、虚拟列表、v-memo、防抖节流等
- uTools 基于 Electron，可利用 Node.js 能力

**建议：**
1. **防抖与节流**：对高频操作（搜索、滚动）进行防抖节流
2. **请求合并**：合并短时间内多次数据库操作
3. **预渲染**：对即将进入视口的内容预渲染
4. **CSS 优化**：使用 `will-change`、`transform` 等优化动画性能
5. **内存管理**：及时清理不再使用的数据和事件监听器

**炫酷效果建议：**
- **手势操作**：支持触摸板手势（如三指滑动）
- **语音控制**：集成语音指令（可选）
- **AI 辅助**：智能预测用户操作，提前加载数据

### uTools 框架特点总结

**优势：**
- 基于 Electron，跨平台桌面应用
- 内置 NoSQL 数据库，支持云同步
- 丰富的 API：窗口、系统、剪贴板、模拟按键等
- 插件生态丰富

**注意事项：**
- 数据库文档冲突问题需要合理设计数据结构
- 插件窗口大小和生命周期管理
- 与 uTools 主进程的通信效率
