# 键盘导航与懒加载冲突问题分析

## 问题描述
在列表使用上下键移动时，超过一页就无法自动把对应的item数据展示到list内，而且数据无法展示齐全。

## 根本原因

### 1. 双重滚动机制冲突
- **Main.vue** 实现了数据懒加载（基于滚动事件）
  - 初始只加载前15条数据（`GAP = 15`）
  - 通过 `scrollCallBack` 监听 `window` 滚动事件到底部时加载更多
  - 代码位置：`[Main.vue#L1181-L1221](../../../src/views/Main.vue#L1181-L1221)`

- **ClipItemList.vue** 使用虚拟滚动组件 `RecycleScroller`
  - 只渲染可见区域的 DOM 元素
  - 通过 `scrollToItem` API 滚动到指定索引
  - 代码位置：`[ClipItemList.vue#L7-L39](../../../src/cpns/ClipItemList.vue#L7-L39)`

### 2. 键盘导航逻辑缺陷
当用户使用上下键导航时：
1. `setKeyboardActiveIndex` 更新 `activeIndex`
2. `scrollActiveNodeIntoView` 调用 `scrollerRef.value.scrollToItem(index)`
3. **问题**：如果目标索引对应的数据尚未被懒加载到 `showList` 中，`RecycleScroller` 会尝试滚动到一个不存在的项目

代码位置：
- 键盘导航注册：`[ClipItemList.vue#L1274-L1288](../../../src/cpns/ClipItemList.vue#L1274-L1288)`
- 滚动逻辑：`[ClipItemList.vue#L1148-L1162](../../../src/cpns/ClipItemList.vue#L1148-L1162)`

### 3. 懒加载触发条件不匹配
懒加载只在 `window` 滚动事件触发时加载：
```javascript
const scrollCallBack = (e) => {
    const { scrollTop, clientHeight, scrollHeight } = e.target.scrollingElement;
    if (scrollTop + clientHeight + 5 >= scrollHeight) {
        // 加载更多
    }
};
```

**问题**：键盘导航使用 `RecycleScroller.scrollToItem` 内部滚动，不会触发 `window` 的滚动事件，因此懒加载不会被触发。

## 应该实现的功能

### 预期行为
1. 使用上下键导航到任意索引时，如果对应数据未加载，应自动触发懒加载
2. 滚动到指定位置后，该位置的数据应该完全可见
3. 虚拟滚动和懒加载应该协同工作，而不是相互冲突

### 修复方案

### 实现方案
在 `ClipItemList.vue` 中监听 `activeIndex` 变化，当接近 `showList` 末尾时触发父组件加载更多数据。

watch(
    () => activeIndex.value,
    () => {
        if (keyboardTriggeredPreview.value) {
            triggerKeyboardPreview();
        }
        // 当键盘导航接近列表末尾时，触发懒加载

#### 4. Main.vue 提取懒加载逻辑
**文件**: `src/views/Main.vue`
**位置**: L1181-L1220

将懒加载逻辑提取为独立函数 `loadMoreData`，供滚动事件和键盘导航共用：
```javascript
// 列表懒加载逻辑提取
const loadMoreData = () => {
    offset.value += GAP;
    const parsed = parseStarFilter(filterText.value);
    let addition = [];
    if (activeTab.value === "collect") {
        const subTab = getCollectSubTab();
        let collectItems =
            subTab === "*全部*"
                ? window.db.getCollects()
                : window.db.getCollectsByTag(subTab);
        collectItems = applyCollectFilters(collectItems, parsed);
        addition = collectItems.slice(offset.value, offset.value + GAP);
    } else {
        const mainBase = getItemsByTab(activeTab.value);
        const mainFiltered = parsed.isStar
            ? mainBase.filter(
                  (item) =>
                      !window.db.isCollected(item.id) &&
                      matchMainTabItem(item, parsed.bodyKeyword),
              )
            : mainBase
                  .filter((item) => !window.db.isCollected(item.id))
                  .filter((item) => matchLockFilter(item))
                  .filter((item) => textFilterCallBack(item));
        addition = mainFiltered.slice(offset.value, offset.value + GAP);
    }
    if (addition.length) {
        showList.value.push(...addition);
    }
};

// 列表懒加载
const scrollCallBack = (e) => {
    const { scrollTop, clientHeight, scrollHeight } =
        e.target.scrollingElement;
    if (scrollTop + clientHeight + 5 >= scrollHeight) {
        loadMoreData();  // 复用提取的函数
    }
};
```

## 修复效果
- 键盘导航（上下键、PageUp/Down、Home/End）在接近已加载数据末尾时自动触发懒加载
- 滚动到任意位置的数据都能正常显示
- 虚拟滚动和懒加载协同工作，不再冲突
- PageUp/PageDown 使用实际容器高度计算 90% 翻页步长，更准确
- Home/End 正确跳转到列表首尾

### PageUp/PageDown 实现细节
**文件**: `src/cpns/ClipItemList.vue`
**位置**: L1176-L1186, L1295-L1314

```javascript
const getPageStep = () => {
    // 在虚拟滚动模式下，使用固定的 item-size 计算
    const itemSize = 60; // 与 RecycleScroller 的 item-size 保持一致
    // 使用实际滚动容器的高度，而不是 window.innerHeight
    const containerHeight = scrollerRef.value?.$el?.clientHeight ||
                            listRootRef.value?.clientHeight ||
                            window.innerHeight ||
                            document.documentElement.clientHeight ||
                            600;
    return Math.max(1, Math.floor((containerHeight / itemSize) * 0.9));
};
```

- PageDown: 向下移动 `getPageStep()` 个项目（约一页的 90%）
- PageUp: 向上移动 `getPageStep()` 个项目（约一页的 90%）
- 使用实际容器高度计算，确保翻页准确

### Home/End 实现细节
**文件**: `src/cpns/ClipItemList.vue`
**位置**: L1315-L1329

```javascript
registerFeature("list-home", () => {
    if (isFocusInSearch()) return false;
    window.toTop();
    return setKeyboardActiveIndex(0, {
        block: "start",
        forceScroll: true,
    });
});
registerFeature("list-end", () => {
    if (isFocusInSearch()) return false;
    return setKeyboardActiveIndex(props.showList.length - 1, {
        block: "end",
        forceScroll: true,
    });
});
```

- Home: 跳转到列表第一项（索引 0）
- End: 跳转到列表最后一项（索引 `showList.length - 1`）
- 使用 `forceScroll: true` 强制滚动到可见区域
