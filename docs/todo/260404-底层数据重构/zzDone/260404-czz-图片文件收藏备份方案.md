# 图片和文件收藏备份方案

## 需求描述
图片收藏和文件，使用自管理的文件路径备份，然后再 uTools 工具内使用文件路径引用形式。

## 当前实现分析

### 1. 当前数据结构
- **图片**: `item.data` 存储 base64 编码的图片数据
- **文件**: `item.data` 存储 JSON 格式的文件路径列表 `[{path: "xxx"}]`
- **收藏**: 直接复制原始数据到 `collectData`，不进行备份

### 2. 当前收藏逻辑
**文件**: `src/global/utoolsDB.js` L307-L364
```javascript
async addCollect(itemId, log = true) {
  // 从数据中查找项目
  let itemToCollect = this.dataCache.data.find(i => i.id === itemId)
  
  // 直接复制原始数据
  itemToCollect = {
    ...itemToCollect,
    collectTime: Date.now(),
    tags: itemToCollect.tags || [],
    remark: itemToCollect.remark || ''
  }
  
  this.dataCache.collectData.unshift(itemToCollect)
}
```

### 3. 当前限制
**文件**: `src/hooks/useClipOperate.js` L147-L150
```javascript
} else if (id === "collect") {
  return item.type !== "file" && !window.db?.isCollected?.(item.id);
}
```
当前**不允许收藏文件类型**。

## 目标实现

### 1. 自管理备份目录
- 位置：`utools.getPath('userData')/clipboard-manager-backup/`
- 子目录结构：
  ```
  clipboard-manager-backup/
  ├── images/          # 图片备份
  │   └── {itemId}.png
  └── files/           # 文件备份
      └── {itemId}/
          ├── original/   # 原始文件
          └── metadata.json
  ```

### 2. 收藏时备份逻辑

#### 图片收藏
1. 检查 `item.type === 'image'`
2. 将 `item.data` (base64) 保存为文件：`backup/images/{itemId}.png`
3. 修改收藏数据：
   ```javascript
   {
     ...itemToCollect,
     data: null,  // 清空 base64
     backupPath: 'clipboard-manager-backup/images/{itemId}.png',
     isBackup: true
   }
   ```

#### 文件收藏
1. 检查 `item.type === 'file'`
2. 解析 `item.data` 获取原始文件路径列表
3. 为每个文件创建备份目录：`backup/files/{itemId}/original/`
4. 复制文件到备份目录
5. 保存元数据：`backup/files/{itemId}/metadata.json`
   ```json
   {
     "originPaths": ["原始路径1", "原始路径2"],
     "backupTime": 1234567890
   }
   ```
6. 修改收藏数据：
   ```javascript
   {
     ...itemToCollect,
     data: JSON.stringify([
       { path: "clipboard-manager-backup/files/{itemId}/original/filename.ext" }
     ]),
     originPaths: ["原始路径1", "原始路径2"],  // 保留原始路径
     backupPath: 'clipboard-manager-backup/files/{itemId}',
     isBackup: true
   }
   ```

### 3. 取消收藏清理逻辑
- 删除备份目录：`backup/images/{itemId}.png` 或 `backup/files/{itemId}/`
- 恢复原始数据结构

### 4. 复制/粘贴逻辑修改
- 检测 `item.isBackup === true`
- **图片**: 从 `backupPath` 读取文件，转换为 base64 复制
- **文件**: 使用 `backupPath` 中的文件路径进行复制

### 5. UI 显示逻辑
- 图片预览：优先使用 `backupPath`，如果不存在则回退到 `thumbnail` 或 `data`
- 文件列表：正常显示，因为 `data` 已经是备份路径

## 实现步骤

### 1. 创建文件备份工具模块
**新文件**: `src/global/fileBackupManager.js`

功能：
- `ensureBackupDir()` - 确保备份目录存在
- `backupImage(itemId, base64Data)` - 备份图片
- `backupFiles(itemId, filePaths)` - 备份文件
- `restoreImage(backupPath)` - 从备份恢复图片数据
- `cleanupBackup(itemId)` - 清理备份

### 2. 修改 utoolsDB.js
- 在 `addCollect` 中调用备份逻辑
- 在 `removeCollect` 中调用清理逻辑
- 添加 `isBackup` 标记

### 3. 修改 useClipOperate.js
- 移除文件收藏限制
- 在收藏操作中调用备份

### 4. 修改复制逻辑
**文件**: `src/utils/index.js`
- `copyOnly` 函数检测 `isBackup` 标记
- 从备份路径读取数据

### 5. 修改图片显示逻辑
**文件**: `src/cpns/ClipItemList.vue`
- `getItemImageSrc` 函数处理 `backupPath`

## 注意事项

1. **存储空间**: 备份文件会占用磁盘空间，需要考虑清理策略
2. **文件存在性**: 复制时需要检查备份文件是否存在
3. **并发处理**: 备份大文件时需要异步处理，避免阻塞
4. **错误处理**: 备份失败时的降级方案（保留原始数据）
5. **迁移兼容**: 已有收藏数据的迁移策略

## 非目标
- 不修改历史数据的收藏逻辑（仅新收藏使用备份）
- 不实现自动清理旧备份的功能
- 不实现备份文件的压缩/去重
