# 底层数据重构原始需求梳理

## 目标: 借助AI实现视觉展示需求

* 数据加载齐全, 不要在 滚动或上下键移动时, 无法把高亮的item展示出来(要自动滚动展示出现), 绝对避免原始数据遗漏无法展示
* 数据移动要及时渲染刷新, pagedown/up 触发窗口单页的90%移动;  按住上下键可触发自动鼠标滚动并自动追随选中最下一条数据(自动滚动且选中)
* 数据展示可使用层叠形式, 叠加但展示第一行数据, 鼠标悬浮即选中, 上下键可在选中的位置为起点切换选中的item
  * 选中的效果: 自动展示全部信息, 置于最顶层, 不要被其他item遮盖或重叠混淆(只需要展示选中的item), 
  * 如果下方的item有数据重叠, 可采用下移方式体现 或 使用 只置顶展示此条, 不要有重叠效果(直接置顶覆盖--正常层叠展示也是不要透明重叠)
* 每个展示item, 时间信息放到第一行, 这样层叠展示时, 时间信息和第一行内容 可直接展示出来
* 鼠标移动时 可以切换选中的item, 键盘上下键移动时临时禁用鼠标效果(移动再重新激活)

## 技术调研与优化建议

### 一、需求缺失分析

基于当前需求梳理，以下方面可能需要补充：

#### 1. 性能边界定义
- **缺失**：未明确数据量级预期（如100条、1000条、10000条）
- **建议**：明确不同数据量级下的性能要求，超过阈值时是否需要采用虚拟滚动

#### 2. 视觉反馈细节
- **缺失**：未定义选中状态的视觉反馈（如高亮颜色、阴影效果、过渡动画）
- **建议**：补充选中状态的样式规范，包括 hover、active、focus 状态

#### 3. 边界条件处理
- **缺失**：未定义空数据状态、加载中状态、错误状态
- **建议**：补充异常场景的UI展示方案

#### 4. 响应式适配
- **缺失**：未考虑不同窗口尺寸下的展示效果
- **建议**：定义窗口高度变化时的层叠间距自适应策略

#### 5. 辅助功能
- **缺失**：未考虑屏幕阅读器等辅助功能
- **建议**：补充 ARIA 标签和键盘导航的无障碍支持

### 二、炫酷效果建议

#### 1. 层叠卡片动效（基于卡片式UI设计最佳实践）
- **微交互**：鼠标悬浮时卡片轻微上浮（transform: translateY(-4px)），配合阴影加深
- **展开动画**：选中卡片展开时使用平滑过渡（transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1)）
- **光影效果**：使用多层阴影打造立体感
  ```css
  box-shadow: 0 2px 8px rgba(0,0,0,0.08), 0 4px 16px rgba(0,0,0,0.12);
  ```
- **玻璃拟态**：选中卡片可使用 backdrop-filter: blur() 实现磨砂玻璃效果

#### 2. 滚动与选中动效
- **平滑滚动**：使用 CSS scroll-behavior: smooth 或 JavaScript 实现平滑滚动
- **选中指示器**：添加左侧高亮条或选中图标指示当前选中项
- **波纹效果**：点击时产生 Material Design 风格的波纹扩散效果

#### 3. 数据加载动效
- **骨架屏**：数据加载时显示骨架屏占位
- **交错动画**：列表项依次淡入（stagger animation），使用 delay 实现级联效果

### 三、刷新率与交互效率优化

#### 1. 虚拟滚动技术（针对大数据量）
- **推荐库**：vue-virtual-scroller（尤雨溪推荐）
- **原理**：只渲染可视区域内的元素，大幅减少DOM节点数量
- **适用场景**：数据量超过500条时启用
- **实现要点**：
  - 预渲染可视区域上下各2-3个item，避免滚动时白屏
  - 动态计算item高度，支持不等高item

#### 2. GPU硬件加速
- **CSS属性**：使用 transform、opacity、filter 触发GPU加速
- **实现方式**：
  ```css
  .card-item {
    will-change: transform, opacity;
    transform: translateZ(0);
  }
  ```
- **注意事项**：
  - 避免过度使用GPU加速，会占用较多内存
  - GPU加速可能影响字体抗锯齿效果

#### 3. requestAnimationFrame优化
- **用途**：替代 setTimeout/setInterval 实现动画
- **优势**：
  - 自动跟随屏幕刷新率（通常60fps）
  - 页面不可见时自动暂停，节省CPU
  - 避免掉帧，保证流畅度
- **应用场景**：
  - 键盘长按连续滚动
  - 平滑滚动动画
  - 卡片展开/收起动画

#### 4. 渲染策略优化
- **v-show代替v-if**：缓存已渲染的DOM，避免重复创建/销毁
- **绝对定位+独立滚动**：
  - 将列表容器设置为绝对定位
  - 使用 z-index 控制显示/隐藏
  - 每个视图拥有独立滚动条
- **防抖与节流**：
  - 滚动事件使用节流（throttle）
  - 搜索输入使用防抖（debounce）

#### 5. 键盘导航优化（基于MDN最佳实践）
- **事件监听**：使用 onkeydown 而非 onkeypress（兼容性更好）
- **键盘鼠标一致性**：确保键盘和鼠标触发相同的状态变化
- **阻止默认行为**：方向键滚动时阻止浏览器默认滚动
- **焦点管理**：使用 tabindex 和 focus() 管理焦点
- **按键重复处理**：不依赖按键重复的一致行为（不同OS/浏览器表现不同）

#### 6. uTools平台特定优化
- **窗口API**：使用 utools.setExpendHeight() 动态调整窗口高度
- **子输入框**：使用 utools.setSubInput() 提供搜索功能
- **暗色模式**：使用 utools.isDarkColors() 检测系统主题

### 四、技术选型建议

| 技术点 | 推荐方案 | 备注 |
|--------|----------|------|
| 虚拟滚动 | vue-virtual-scroller | 尤雨溪推荐，功能完善 |
| 动画库 | @vueuse/motion 或 CSS transition | 轻量级优先 |
| 状态管理 | Pinia（如需复杂状态） | 当前项目可能不需要 |
| 滚动优化 | 原生scroll + requestAnimationFrame | 避免引入额外依赖 |

### 五、性能测试指标

- **目标帧率**：60fps（16.67ms/帧）
- **滚动延迟**：<100ms
- **选中响应**：<50ms
- **内存占用**：大数据量时内存增长<50MB
- **测试工具**：Chrome DevTools Performance、Lighthouse
