# Clipboard 修复计划

## 目标

把当前 `Clipboard` 列表行为收敛到一套稳定模型，优先修复以下问题：

- 键盘移动一步一项
- 自动滚动行为可预测
- 删除后高亮和列表状态稳定
- 展示列表和真实数据映射一致
- 虚拟滚动定位可靠

## 总体策略

本次修复不建议继续叠加补丁式判断，而应先收敛状态责任边界。

核心原则：

1. `activeIndex` 只允许一处主控
2. 所有键盘导航都基于同一份“展示列表”
3. 所有删除后的恢复都走同一条路径
4. 虚拟滚动只保留一种可信定位方式

## 分阶段计划

### 第一阶段：止血，先消除最明显的错乱

目标是让行为先恢复到“可用且稳定”。

计划：

1. 移除上下键的双重驱动
   - 保留 `registerFeature("list-nav-up/down")` 或保留 `unifiedKeyHandler` 中的一套
   - 不允许两套逻辑同时推进 `activeIndex`

2. 暂停或简化自动滚动
   - 在基础导航稳定前，先禁用 `startAutoScroll` 的自动推进
   - 或改成只负责长按重复，不负责首帧推进

3. 修复收藏页强制删除中的未定义变量
   - 补齐 `currentActiveIndex`
   - 保证该路径不会抛出运行时错误

4. 去掉删除后父组件和子组件重复恢复高亮的竞争
   - 先选定一处作为删除后索引恢复的唯一入口
   - 建议先由 `ClipItemList.vue` 统一负责

### 第二阶段：统一展示层语义

目标是让“用户看到的列表”和“代码用于导航的列表”完全一致。

计划：

1. 明确单一展示数组
   - 例如统一使用 `currentShowList`
   - 任何导航、删除、高亮、步进都只认这一份数组

2. 审核以下逻辑是否仍直接依赖 `showList`
   - 点击推进
   - 多选区间选择
   - 删除恢复
   - 懒加载后的跳转
   - 快捷复制 `alt+1~9`

3. 对 `collectBlockList + showList` 的拼接场景建立统一索引规则
   - 当前索引必须始终面向展示数组
   - 需要访问真实数据时通过 item.id 反查，不再依赖局部数组索引

### 第三阶段：重做虚拟滚动定位

目标是让“滚到目标项”真正可靠。

计划：

1. 以 `DynamicScroller.scrollToItem` 作为唯一主路径
   - 优先用组件 API 定位
   - 不再把“半屏滚动”当作正常 fallback

2. 减少基于估算 `itemSize` 的手工滚动
   - 当前 `DynamicScroller` 是不等高场景
   - 用固定 `44px` 估算仅适合作为极弱兜底

3. 如果需要 fallback，只保留“确保目标可见”的策略
   - 不要再限制单次最多滚半屏
   - 否则会再次制造“目标项没进来”的假象

4. 检查 `nextTick` 和虚拟列表渲染时机
   - 需要在数据变化后等待 scroller 完成一轮更新
   - 再执行目标项定位

### 第四阶段：重构删除恢复模型

目标是删除任何项后，高亮位置规则稳定且可解释。

建议统一规则：

- 删除当前项后，优先落到原位置上的下一项
- 如果原位置已越界，则落到新的最后一项
- 多选批量删除后，同样按删除前 anchorIndex 归一化

计划：

1. 在删除前只记录两类信息
   - `anchorIndex`
   - 当前展示列表中被高亮项的 `id`

2. 删除完成并刷新展示列表后，只执行一次恢复
   - 优先按 `id` 找回
   - 找不到再按 `anchorIndex` 归一化

3. 移除重复的恢复机制
   - 不同时保留父组件同步赋值
   - 不同时保留父组件 `setTimeout`
   - 不同时保留子组件 watcher 的第二套修正

### 第五阶段：回归验证

修复后至少需要回归以下场景：

1. 普通列表上下键移动
2. 长按上下键自动滚动
3. PageUp/PageDown 翻页
4. 普通删除和强制删除
5. 多选删除
6. 收藏页删除和取消收藏
7. `*` 搜索下顶部收藏块与普通列表混排
8. 鼠标左键复制后自动跳下一项
9. 右键打开抽屉后自动跳下一项
10. 懒加载边界处继续下移

## 建议实施顺序

建议按下面顺序落地，避免边改边引入新冲突：

1. 修复未定义变量和双重导航
2. 统一删除恢复责任
3. 统一展示数组语义
4. 收敛虚拟滚动定位
5. 再恢复或重做自动滚动体验

## 预期结果

完成后，列表行为应满足：

- 任意一次方向键操作都只产生一次确定移动
- 删除后高亮位置可预测
- 顶部收藏块不会破坏主列表索引
- 虚拟滚动不会再出现“只滚半页但项进不来”
- 列表导航、点击推进、懒加载三者能共存而不互相覆盖
