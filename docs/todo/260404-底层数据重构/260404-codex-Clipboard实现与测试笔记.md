# Clipboard 实现与测试笔记

## 背景

本笔记用于记录本轮 `Clipboard` 列表问题处理中：

- 已实施的代码调整
- 用户实际测试反馈
- 当前确认有效和无效的判断
- 后续继续修复时的约束

目标是为后续继续修复提供连续上下文，避免再次误判链路或重复踩坑。

## 本轮实施内容

### 第一阶段实施

已做过的调整：

1. 收敛删除后索引恢复
   - `src/views/Main.vue`
   - 删除父组件里“同步改一次 activeIndex，再异步再改一次”的双重恢复
   - 改成主要依赖一次统一恢复逻辑

2. 修复收藏页强制删除中的明显错误
   - `currentActiveIndex` 在收藏页强制删除路径中原本未定义
   - 已补齐并修正

3. 尝试收敛方向键双重推进
   - 初始判断是方向键有两套推进源：
     - `HotkeyProvider -> hotkeyRegistry -> list-nav-up/down`
     - `ClipItemList.vue -> unifiedKeyHandler -> startAutoScroll`
   - 第一轮曾尝试直接削弱 `unifiedKeyHandler`

### 第一阶段实施后暴露的问题

出现了新的回归：

- 方向移动一度完全失效

这说明：

- 当前实际可用链路不能简单理解为“hotkey feature 已完全接管方向键”
- 直接删减 `unifiedKeyHandler` 是过度收敛

### 第二次修正

随后做了可用性恢复：

1. 恢复 `ArrowUp/ArrowDown` 的实际移动能力
2. 保留 `unifiedKeyHandler` 继续参与方向移动
3. 让 `list-nav-up/down` 不再处理物理方向键本身，避免和 `unifiedKeyHandler` 重复推进

该阶段之后，方向移动恢复可用。

## 用户测试反馈

用户最新测试结果：

- 现在移动效果有了
- 最起码不会一直隐藏选中 item
- 但是画面一直滚动，渲染效果不好
- 在可以完整展示选中项时，不需要进行页面滚动
- 只有顶部/底部无法展示选中项时，再进行半页提前滚动
- 按住上下键时，希望翻页同时自动居中选中 item

## 当前确认有效的结果

截至这份笔记，已经确认有效的点：

1. 方向移动已恢复
2. 选中项不再总是长期处于被隐藏状态
3. 删除恢复逻辑已较之前更收敛
4. 收藏页强制删除的明显实现错误已修正

## 当前仍然存在的问题

### 1. 单步移动时滚动过多

当前问题不是“不能动”，而是：

- 单步上下移动时仍然容易触发页面滚动
- 即使当前视口完全能容纳选中项，也发生滚动

这会导致：

- 视觉抖动明显
- 虚拟滚动不断触发重排和重绘
- 用户体感差

### 2. 单步移动缺少“无需滚动”的判断

当前缺少的不是“滚动能力”，而是“是否应该滚动”的判定。

正确需求已经明确：

1. 选中项若已完整可见，则不滚动
2. 只有当选中项即将被顶部或底部裁掉时，才触发预滚动
3. 这种预滚动应优先采用半页滚动，而不是每次都强行对齐或居中

### 3. 长按上下键的行为仍未达到目标

用户期望：

- 长按上下键时，列表应呈现翻页式推进
- 同时应自动将当前选中项居中

也就是说：

- 单步移动和长按移动应当是两种不同的滚动策略
- 单步移动优先“最小滚动 / 不滚动”
- 长按移动优先“分页推进 / 自动居中”

## 当前对问题的更准确理解

经过这轮实现和测试，当前应当采用如下判断：

### 1. 方向移动主链路不能再被误删

当前代码中，物理方向键的可用性仍与 `unifiedKeyHandler` 有强关联。

因此后续继续修时：

- 不要再次直接移除 `unifiedKeyHandler` 的方向键职责
- 应该在保留主链路的前提下，细分单步滚动与长按滚动行为

### 2. 当前核心问题已从“不能移动”转为“滚动策略错误”

也就是说，下一步不是继续修“移动失效”，而是修：

- 单步导航的滚动阈值
- 半页预滚动策略
- 长按翻页并居中的策略

### 3. 删除问题仍需继续观察，但优先级暂时低于滚动体验

当前用户最新反馈聚焦在：

- 滚动太频繁
- 视觉效果不好
- 长按翻页不符合预期

因此下一轮更应优先修滚动策略，而不是继续大改删除逻辑。

## 后续实施约束

后续继续修复时，应遵守以下约束：

1. 不再大范围改动方向键主链路
2. 不再直接尝试“删除某整套导航机制”
3. 先保住现有“能移动”状态
4. 滚动策略只围绕以下目标精修：
   - 可见则不滚
   - 不可见时半页预滚
   - 长按时翻页并自动居中

## 建议的下一轮实现方向

建议下轮从滚动策略层面拆成两套：

### 单步移动

目标：

- 如果选中项已完整可见，不滚动
- 如果选中项即将超出顶部/底部边界，再触发半页预滚动

### 长按移动

目标：

- 不再沿用单步移动的逐项强滚动
- 使用分页推进
- 每次翻页后将选中项自动居中

## 结论

本轮处理后，问题状态已经发生变化：

- 之前是“移动异常甚至失效”
- 现在是“移动恢复，但滚动策略不符合要求”

这说明后续修复应该进入“滚动体验精修阶段”，而不是继续做大范围链路收敛。
